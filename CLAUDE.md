# 작업 메모리 — EN_vocab / NoEnglish (한→영 어휘 학습 PWA)

## 작업 원칙 (★가장 중요★)
- **수정할 때 내 마음대로 판단해서 사용자의 디자인을 롤백/제거하지 말 것.**
  무언가 바꾸기 전에 반드시 **기획서(`docs/PLAN.md`)와 파이프라인(`pipeline/`)을 먼저 확인**하고 기존 설계를 존중한다.
  임의 판단으로 되돌렸다가 다시 만드는 것은 **토큰·시간 낭비**다. (예: 홈 뱃지 트랙을 마음대로 제거했다가 다시 복원한 사례 — 반복 금지.)
- 요청받은 것만 정확히, **하나씩 실제 브라우저 렌더로 검증**하고 넘어간다. "됐다"고 말하기 전에 반드시 렌더 확인.
- 애매하면 추측해서 큰 변경을 하지 말고 먼저 확인한다.

## 대화 스타일
- 사용자와 대화할 때는 항상 **존댓말**을 사용한다.

## 디자인 규칙
- **이 앱에는 절대 이모지를 넣지 않는다.** (픽토그램/컬러 이모지 금지. UI 텍스트·안내문·알림 어디에도.)
  - 체크(✓)·닫기(✕)·화살표(→ › ↔) 같은 기능적 텍스트 글리프는 예외.
- **섹션 타이틀은 `.sec` 클래스(17px/800) 공용 — 새 화면 만들 때도 이거 재사용.** (2026-07-15, 13px가 너무 작다는 지적으로 26px까지 확대했다가, 이번엔 너무 커졌다는 지적으로 17px로 재조정 — 26px의 2/3.)
- **형태(border-radius)는 네모(0) / 원형(50%, 정사각형 요소만) / 알약(999px, 태그·칩·버튼류) 셋 중 하나만.** 8~24px 같은 중간 라운드 금지 — 아날로그 노트 느낌 컨셉(2026-07-15). `--radius` 변수도 0.
- **메뉴 좌측 아이콘 슬롯(`step-ico`)은 원형+블랙 배경(`.step-ico.round.black`) 공용.** 오늘의 루틴, 학습 탭(단어카드/듣기세션/레벨테스트), 사전 탭 메뉴 전부 이 조합 — 새 메뉴 추가할 때도 재사용.
- **탭바**: 잉크 블랙 배경, 비활성 아이콘은 밝은 크림, 활성 탭마다 형광펜 하이어라키에서 서로 다른 색(학습=초록·사전=노랑·홈=핑크·회화=파랑·보관함=오렌지). 홈 전용 튀어나오는 원형 FAB 없음 — 모든 탭 동일 레벨. 아이콘 26px/폰트 11px(기본 24px·10px의 110%).
- **연속일 점 색상은 뱃지 티어와 무관한 별도 하이어라키** — 핑크=위험(연속 0~2일) · 노랑=보통(3~6일) · 초록=잘하고있음(7~29일) · 파랑=매우좋음(30일+). `streakHl()` 참고. 점 색은 `-dark` variant가 아니라 형광펜 베이스(일반) 톤 사용 — 진한 쪽은 너무 어둡다는 지적으로 되돌림(2026-07-16).
- **웜그레이 하이어라키(2026-07-16 재구성)**: 새 톤 3개를 따로 만들지 않고 이미 있는 배경/텍스트 톤을 밝은쪽·진한쪽 끝으로 재사용 — 밝은쪽 `--line`, 진한쪽 `--muted`, 그 사이를 잇는 중간톤만 `--warmgray`로 신설(#B9AF99). `--accent-soft` 골드톤과 짝을 이루는 "강조하되 안 노란" 옵션. mid(`--warmgray`)는 설정 보유 포인트 배경 + 듣기속도 활성값 + 회화 피드백 카드에 공용으로 씀. (예전엔 `--warmgray-light/-dark`를 별도 신설했었는데, 컬러스와치 리뷰 후 기존 토큰 재사용으로 정리 — 새 토큰을 늘리기 전에 이미 있는 배경/텍스트 톤으로 채울 수 있는지 먼저 볼 것.)
- **복습뱃지(`.tag.due-t0~3`) 잉크 위계는 가장 진한 단계도 50% 캡.** (과거 75%에서 완화 — 너무 진하다는 지적.)
- **카운트다운(3·2·1) 오버레이는 카드가 아니라 화면 전체(탭바 포함)를 10% 딤.** `.dict-countdown-overlay`가 `position:fixed`인 이유 — `absolute`로 두면 카드 안에서만 딤돼서 요구사항과 다름.
- **상단바 "연속N일" pill과 설정 버튼(`.avatar-btn`)은 검정 스트로크(`border:1px solid var(--ink)`)** — 검색창(`.search`)과 동일한 두께/색으로 통일.
- **홈 "학습 캘린더"는 실제 이번 달 요일에 맞춘 달력**(예전 잔디/히트맵 스트립에서 교체, 2026-07-16) — `Home`의 `calendar` useMemo가 `Date` 계산으로 요일·일수를 맞춤. 학습량 표시(칸 배경 진하기)는 아직 더미 데이터(실 학습 로그 연동 전).
- **회화 대화방(`TalkRoom`)은 페이지 배경(`--bg`)보다 밝은 `--surface`에 검정 테두리를 두른 별도 "채팅창"** — 그 안에서만 말풍선이 오가서 카톡/문자 앱처럼 또렷하게 구분되도록(2026-07-16). 봇 말풍선 옆엔 페르소나 색 원형 미니 아바타(`.talk-avatar-sm`). 방 입장 시 Gemini 왕복 없이 `persona.intro`(고정 영문 자기소개)를 즉시 보여주고 그 자리에서 TTS 재생. 인풋은 `autoFocus`로 키보드 자동 표시. 음성 입력은 브라우저 내장 `SpeechRecognition`(서버 불필요) — 미지원 브라우저에선 버튼 자체를 숨김(`SpeechRecognitionAPI` 존재 여부로 분기).

## 오디오
- **모바일 자동재생 언락 패턴**: 세션 자동재생(딕테이션·회화 대화방 등)처럼 `setTimeout`/비동기 뒤에 소리를 재생하면 사용자 제스처와 끊겨 모바일(특히 iOS Safari)에서 무음 재생 실패한다. `unlockAudio()`가 `_sharedAudio` 엘리먼트를 한 번 재생해 언락해두면 이후 타이머 기반 재생도 허용된다 — `speakText()`의 모든 TTS 재생이 이 엘리먼트를 재사용. **`unlockAudio()`는 반드시 실제 오디오가 필요한 진입점(듣기 세션 시작, 회화 대화방 입장 등)의 클릭 핸들러 안에서만 호출할 것 — `document` 전역 `pointerdown` 같은 "앱 아무 데나 첫 터치"에 걸어두면 안 됨.** 예전에 전역으로 걸어놨다가, 무음이라도 `<audio>` 재생·음성합성 호출 자체가 iOS에서 백그라운드 음악(다른 앱)의 오디오 세션을 가로채 정지시키는 버그가 있었다(홈 화면 아무 곳이나 눌러도 발생, 2026-07-16 수정). 새 오디오 재생 경로를 추가할 때 `new Audio()`를 따로 만들지 말고 `_sharedAudio`를 재사용하고, 지연 재생이 필요하면 그 흐름을 시작하는 클릭 핸들러에서 `unlockAudio()`를 명시적으로 호출할 것.

## 문구 규칙
- 안내(업데이트 안내 등) 문구는 핵심/중요 부분을 볼드로 강조한다. (`WHATS_NEW`는 `**중요부분**` 마크업, `renderBold`가 `<b>`로 렌더.)

## 버전 표기
- 기능을 수정/배포할 때마다 `index.html`의 `APP_VERSION`을 갱신하고, `<head>`의 `?v=` 캐시 무효화 태그(manifest/words.js/rat_run.js)도 **같은 버전으로 함께** 갱신한다.
- `APP_BUILD`는 작업 당일 날짜(YYYY-MM-DD).
- **`version.json`의 `v` 값도 APP_VERSION과 항상 같이 갱신할 것.** PWA는 새로고침 제스처가 없어서, 앱이 이 파일을 캐시 없이 주기적으로 읽어 배포 감지 배너(`.update-banner`)를 띄운다 — 안 올리면 사용자에게 새 버전 알림이 안 감.

## 데이터 파이프라인 메모
- 시드는 `pipeline/`에서 생성 → `pipeline/merge_cefrj.js`가 `words.js`로 병합. `words.js`는 직접 수정 금지(배치/보강 파일 수정 후 재빌드).
- 다의어 senses는 `pipeline/senses_parts/*.json`(에이전트별 분할)을 merge가 합침. WordNet 뼈대는 `pipeline/wordnet_senses.json`.
- **다의어 배치 실행 개수 제한**: 한 세션에서 배치를 **2개 완료하면 자동으로 다음 배치를 이어서 실행하지 말 것.**
  사용자가 "더 해" 등으로 명시적으로 요청하기 전까지는 멈추고 기다린다. (세션 리밋 대비 보수적 실행 원칙과 별개로,
  배치 개수 자체를 사용자 통제하에 둔다.)

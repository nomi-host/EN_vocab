# 작업 메모리 — EN_vocab / NoEnglish (한→영 어휘 학습 PWA)

## 작업 원칙 (★가장 중요★)
- **수정할 때 내 마음대로 판단해서 사용자의 디자인을 롤백/제거하지 말 것.**
  무언가 바꾸기 전에 반드시 **기획서(`docs/PLAN.md`)와 파이프라인(`pipeline/`)을 먼저 확인**하고 기존 설계를 존중한다.
  임의 판단으로 되돌렸다가 다시 만드는 것은 **토큰·시간 낭비**다. (예: 홈 뱃지 트랙을 마음대로 제거했다가 다시 복원한 사례 — 반복 금지.)
- 요청받은 것만 정확히, **하나씩 실제 브라우저 렌더로 검증**하고 넘어간다. "됐다"고 말하기 전에 반드시 렌더 확인.
- 애매하면 추측해서 큰 변경을 하지 말고 먼저 확인한다.

## 대화 스타일
- 사용자와 대화할 때는 항상 **존댓말**을 사용한다.

- **음성 입력(STT)은 브라우저 내장 `SpeechRecognition`이 아니라 `MediaRecorder`+서버 전사.** iOS Safari는 `SpeechRecognition`/`webkitSpeechRecognition` 자체가 없어서 예전 방식은 버튼이 조용히 사라졌었다(2026-07-16 교체). `getUserMedia`+`MediaRecorder`(iOS Safari 14.3+ 지원)로 녹음 → base64로 Worker `/stt`에 전송 → Gemini 멀티모달(`inline_data` 오디오)로 전사한 텍스트를 인풋에 채움. 지원 여부 분기는 `STT_SUPPORTED`(`getUserMedia`+`MediaRecorder` 존재 확인)로, `SpeechRecognitionAPI` 체크가 아님.
- **Worker 프록시 인증(2026-07-16 추가)**: "쓴 적 없는데 Gemini 쿼터 초과" 문의로 확인해보니, `ALLOWED_ORIGINS`가 비어있는(전부 허용) 상태 + `PROXY_BASE` URL이 `index.html`에 평문으로 박혀있어(누구나 사이트 열어보면 페이지 소스로 확인 가능) — CORS의 Origin 헤더는 서버-서버 요청에서 위조 가능해 실질적 보호가 안 됨 — 사실상 인증 없는 공개 엔드포인트였음. `PROXY_SECRET`(`index.html`)/`APP_SHARED_SECRET`(Worker 시크릿, 반드시 같은 값)을 `X-App-Secret` 헤더로 주고받아 불일치 시 403 — 클라이언트 코드 노출이라 완벽한 인증은 아니지만 무작위 스캐너/봇 남용은 막음(`worker/README.md` 참고). `ALLOWED_ORIGINS`도 실제 배포 주소로 좁혀둘 것.
- **Gemini 모델은 단일 지정이 아니라 `MODEL_FALLBACKS` 폴백 체인(2026-07-16 최종).** Worker가 후보 목록(텍스트: `gemma-4-26b-a4b-it` → `gemini-3-pro-preview` → `gemini-2.5-flash-lite` → `gemini-2.0-flash-001` → `gemini-2.0-flash`)을 순서대로 시도, 404/429는 다음 후보로 자동 전환 — Google이 모델을 또 막아도 앱은 계속 동작. `GEMINI_MODEL` 대시보드 Variable은 만들지 말 것(설정하면 최우선 후보로 끼어들며, 옛날 값이 남으면 코드보다 우선 적용되는 함정 있음 — 실제로 한 번 헛돌았음). 아래는 그 이전 단일 모델 시절의 기록:
- **Gemini API 무료 티어의 2026년 실체(웹 서치로 확인, 2026-07-16 — 추측으로 여러 번 헛돈 뒤 정리).** ① 무료 티어는 **결제(선불 Cloud Prepay 포함)가 연결 안 된 프로젝트에서만** 존재 — 결제를 붙이는 순간 그 프로젝트의 무료 티어는 소멸하고 전 호출이 과금 대상("billing trap", 02웍스 프로젝트가 이 상태 + 크레딧 0원이라 전부 거부됐음). ② 무료 티어는 **현세대 Flash 계열만** — Pro 모델은 2026-04-01부터 API 유료 전용(그래서 `gemini-3-pro-preview`가 무료 키에서 429). ③ `gemini-2.5-flash`/`-lite`는 공식 종료일(2026-10-16) 전인 **2026-07-09부터 신규 사용자에게 조기 404** — 전 세계적 사건(Google 포럼 다수 보고), 계정 문제 아님. ④ Cloudflare Workers 경유 호출은 홍콩 등 미지원 지역으로 라우팅될 수 있어 무료 티어가 위치 거부(400 FAILED_PRECONDITION)될 수 있음 — 알려진 문제. **결론: 결제 미연결 프로젝트의 키 + 현세대 Flash(`gemini-3.5-flash` 등)가 무료 경로**이고, 폴백 체인 맨 앞에 반영됨. 같은 증상 재발 시 추측 말고 웹 서치부터 할 것.
- **클라이언트 직접 호출 플러밍(`GEMINI_DIRECT_KEY`, index.html)은 폐기(2026-07-17), 값은 항상 빈 문자열.** 위 ④의 위치 문제를 피하는 우회로로 넣었던 건데, 내장 키(AQ. 형식)가 계정 단위 버그로 401만 반환해 앱이 계속 실패했음 — Vercel 프록시(서울 리전 고정, 위치 문제 이미 해결됨)로 완전 전환하면서 이 우회로 자체가 불필요해짐. 코드(`getGeminiKey()`)는 남겨뒀지만 **키를 다시 채우지 말 것** — 프록시(`api/gemini.js`/`api/stt.js`)가 정상 경로다.
- **Gemini API 키 발급은 반드시 AI Studio(`aistudio.google.com/apikey`)에서만 — Cloud Console Credentials 페이지에서 만든 Standard 키는 애초에 Gemini API에 바인딩이 안 됨(2026-07-17 확인).** Gemini API는 "서비스 계정에 바인드된 Auth 키"만 허용해서, Cloud Console에서 API 제한사항 체크박스를 누르면 "이 API에는 서비스 계정에 바인드된 API 키를 사용한 인증이 필요합니다"로 비활성화되어 있음 — 즉 Cloud Console 경로는 막다른 길. 이 바인딩을 자동으로 처리해주는 게 AI Studio 발급 플로우라 `AQ.`로 시작하는 키가 나오는 게 정상. (TTS는 이 제약이 없는 별개 API라 Cloud Console의 Standard `AIza` 키로 계속 정상 작동.) 만약 AQ. 키가 인증 자체는 되는데 401만 반복되면 코드 문제가 아니라 Google 쪽 계정/프로젝트 단위 버그일 수 있음(Google 개발자 포럼에 유사 사례 다수) — 그때는 AI Studio에서 **새 프로젝트**로 키를 다시 발급받는 게 가장 잘 먹히는 우회법.
- **Gemini 3.x 세대(`gemini-3.5-flash` 등)는 `maxOutputTokens`가 "thinking" 토큰과 합산 예산이라, 짧게 잡으면(예: 300) thinking이 대부분을 먹어치우고 실제 답변이 문장 중간에서 끊겨서 나옴(2026-07-17, SOS 번역이 "It was so frustrating because it hasn"처럼 잘림 — 에러 없이 조용히 truncate라 알아채기 어려움).** `api/gemini.js`·`api/stt.js`·`index.html`의 `geminiDirect`/`transcribeAudio` 전부 1024(텍스트)/500(STT)로 상향. 같은 모델 계열 쓰는 한 이 예산으로 부족하면 더 올릴 것 — thinkingConfig로 끄는 방법도 있으나 `gemini-3-pro-preview`처럼 thinking 끄기를 거부하는(최소 128 요구) 모델이 폴백 체인에 섞여있어 일괄 적용은 위험, maxOutputTokens 상향이 더 안전.
- **(과거 기록) Gemini 모델 기본값은 `gemini-3-pro-preview`.** 위 인증 이슈와는 별개로, 실제 사용량이 0인데도 429가 계속 뜨는 걸 사용자가 Google AI Studio "모델별 비율 제한" 화면 스크린샷으로 확인해줌 — `Gemini 2 Flash`(=`gemini-2.0-flash`) RPM 한도가 이 계정에서 **0/0**으로 배정돼 있었음(2026-07-16). 그다음 시도한 `gemini-2.5-flash`, `gemini-2.5-flash-lite` 둘 다 models.list엔 stable로 나오는데도 실제 호출하면 똑같이 `404 "no longer available to new users"` — **즉 models.list에 나온다고 실제로 호출 가능한 게 아님**, 이 계정(신규 발급 키)은 2.x 세대 모델 전체가 막혀있는 것으로 보임. 최신 세대인 `gemini-3-pro-preview`(models.list에서 generateContent 지원 확인)로 바꿔서 해결. **모델명이 계정/시점마다 다를 수 있고 2.x는 이 계정에서 전멸이므로, 같은 증상 재발 시 2.x로 또 추측하지 말고 3.x 이상부터 시도하거나 models.list로 직접 확인할 것.** `worker/src/worker.js`·`worker/wrangler.toml` 기본값 교체 완료. **대시보드의 `GEMINI_MODEL` Variable이 코드 기본값보다 항상 우선**하므로, 코드를 고쳐도 대시보드 값이 옛날 것으로 남아있으면 적용 안 됨 — 실제로 이 때문에 한 번 헛돌았던 적 있음(2026-07-16), 모델 문제 재발 시 반드시 대시보드 값부터 확인할 것.

## 오디오
- **모바일 자동재생 언락 패턴**: 세션 자동재생(딕테이션·회화 대화방 등)처럼 `setTimeout`/비동기 뒤에 소리를 재생하면 사용자 제스처와 끊겨 모바일(특히 iOS Safari)에서 무음 재생 실패한다. `unlockAudio()`가 `_sharedAudio` 엘리먼트를 한 번 재생해 언락해두면 이후 타이머 기반 재생도 허용된다 — `speakText()`의 모든 TTS 재생이 이 엘리먼트를 재사용. **`unlockAudio()`는 반드시 실제 오디오가 필요한 진입점(듣기 세션 시작, 회화 대화방 입장 등)의 클릭 핸들러 안에서만 호출할 것 — `document` 전역 `pointerdown` 같은 "앱 아무 데나 첫 터치"에 걸어두면 안 됨.** 예전에 전역으로 걸어놨다가, 무음이라도 `<audio>` 재생·음성합성 호출 자체가 iOS에서 백그라운드 음악(다른 앱)의 오디오 세션을 가로채 정지시키는 버그가 있었다(홈 화면 아무 곳이나 눌러도 발생, 2026-07-16 수정). 새 오디오 재생 경로를 추가할 때 `new Audio()`를 따로 만들지 말고 `_sharedAudio`를 재사용하고, 지연 재생이 필요하면 그 흐름을 시작하는 클릭 핸들러에서 `unlockAudio()`를 명시적으로 호출할 것.

## 버전 표기
- 기능을 수정/배포할 때마다 `index.html`의 `APP_VERSION`을 갱신하고, `<head>`의 `?v=` 캐시 무효화 태그(manifest/words.js/rat_run.js)도 **같은 버전으로 함께** 갱신한다.
- `APP_BUILD`는 작업 당일 날짜(YYYY-MM-DD).
- **`version.json`의 `v` 값도 APP_VERSION과 항상 같이 갱신할 것.** 앱이 이 파일을 캐시 없이 최초 로드 시 + 포그라운드 복귀 시(PWA를 껐다 켤 때 포함) 확인해서 APP_VERSION과 다르면 **배너/버튼 없이 바로 강제 새로고침**한다 — 안 올리면 사용자가 새 버전을 못 받음.
- **업데이트 감지는 배너+수동 버튼이 아니라 자동 강제 새로고침(2026-07-16 변경).** 이전엔 `.update-banner`에 "새로고침" 버튼을 띄워 사용자가 직접 눌러야 했는데, "버튼 없애고 PWA 껐다 켜면 강제 새로고침되게" 요청으로 배너를 완전히 제거하고 `App`의 버전 체크 effect가 최초 마운트 시 + `visibilitychange`(visible) 시 자동으로 `location.reload()` 대신 `?_=타임스탬프` URL 강제 이동을 실행하도록 변경. 5분 주기 폴링은 제거(활성 세션 중 갑자기 새로고침되어 입력 중이던 내용이 날아가는 걸 피하기 위해 — 재진입 시점에만 체크).
- **`index.html` 안의 `<meta http-equiv="Cache-Control">`은 실제 HTTP 캐시에 안 먹히는 경우가 많음**(특히 iOS 홈화면 standalone 앱) — 진짜 캐시 제어는 배포 루트의 `_headers` 파일(Cloudflare Workers Assets가 읽음)이 담당한다. `index.html`/`manifest.json`/`version.json`은 여기서 `no-cache, no-store, must-revalidate`로 강제 — 이 파일들 캐시 정책을 바꿀 땐 메타 태그가 아니라 `_headers`를 고칠 것.

## 학습/사전 UX 메모
- **단어카드 유의어 정답 유도(`CardSession`)**: 유의어(예 "on time")를 입력하면 오답 처리하지 않고 넛지("'on time'도 맞아요! 그런데 같은 뜻으로 'p'로 시작하는 단어도 있어요") + 첫글자·글자수 힌트를 자동으로 연다. 데이터엔 언더스코어("on_time")로 저장되는데 사용자는 공백("on time")으로 치므로 비교 시 `normSyn`(공백/언더스코어 통일)로 정규화 필수 — 안 하면 유의어 인식이 안 돼 그냥 오답 처리됨(2026-07-17 수정). 유의어 칩 표시도 `_`→공백.
- **카드 위 "뜻"은 항상 보이는 게 아니라 힌트 버튼으로 여는 방식 — 열면 품사(한글)+뜻(2026-07-17 재변경).** 힌트 행에 `[뜻]` 버튼(cloze 전용, 글자수·첫글자 앞)을 누르면 `.qsofthint`에 `meanText`(`wordOf(it.word).pos`→POS_KO로 매핑한 품사 · `it.ko`, 예 "명사 · 의회, 협의회")를 표시. `hintMean` state, `toggleMean`이 켜며 다른 힌트처럼 `hintUsed`(재출제) 처리. spell 모드는 `.qko`가 이미 단어 뜻을 크게 보여줘 `[뜻]` 버튼도 qsofthint도 안 냄. (그 직전엔 뜻을 항상 옅게 노출했었는데, 사용자가 "뜻 열기(앞에 품사도)"로 바꿔달라고 해서 온디맨드로 변경 — 문맥 추측 먼저, 막히면 뜻 열기.) 예전 `softHint`(빈도/격식 근사 문구) 함수는 제거됨.
- **`makeBlank` 굴절 매칭 대폭 강화 — 예문 빈칸 실패 항목 310→7로 축소(2026-07-17).** 예전엔 어간 접두 휴리스틱만이라 apply→applied(y→i)·be→was·leaf→leaves 같은 굴절과 back up→"backed me up" 같은 분리형 구동사를 놓쳐 spell로 빠졌음. `IRREG`(불규칙 동사·복수 명사 token→base 역인덱스)+규칙 역굴절(`lemmasOf`: y→i·자음중복·e탈락·복수)+`formsMatch`(원형↔굴절 양방향, 예전 접두 휴리스틱도 폴백 유지해 기존 매칭 보존)로 재작성. 다단어는 연속 우선, 안 되면 목적어 낀 분리형("backed me up")도 최대 2단어 간격까지 구간 전체를 빈칸(정답에 목적어 포함 — cloze는 예문 한국어 뜻을 통째로 주므로 목적어 유추 가능). 하이픈 합성어(well-known)는 조각만 빈칸. 남은 7개(chewing gum·impersonator·keeper·light·seller·whittle down·wreck)는 닫힌 합성어에 단어가 묻혀있거나(zookeeper) 파생형(impersonations)이라 예문 자체가 부적합했는데, **예문을 단어가 그대로 쓰이는 문장으로 새로 써서 전부 회복 → noblank 0(2026-07-17).** 예문은 `pipeline/batches/*.json`(keeper/wreck/seller/impersonator/chewing gum) 및 `pipeline/senses_parts/*.json`(light#0=00_pilot, whittle down=IDIOM_02) 수정 후 `node pipeline/merge_cefrj.js`로 재빌드(words.js 직접 수정 금지 원칙 준수).
- **단어카드는 무조건 예문 안 빈칸(cloze)만 — spell 폴백은 세션 풀에서 제외(2026-07-17).** 예문에 타깃 단어(어간)가 없어 `makeBlank`가 `null`을 주던 항목은 예전엔 `CardSession`에서 `mode="spell"`(뜻 크게+영단어 직접 입력)로 나왔는데, 사용자가 "단어카드는 무조건 예문 빈칸이어야 한다"고 해서 `canBlank(it)` 필터로 세션 풀에서 아예 뺌. 적용 지점: `freshPool`(신규), `pickSession`의 `due`(복습), 내 단어장 리스트 학습(`studyItems`). 데모 복습 8키는 전부 cloze 가능(검증)해서 "복습 8"은 유지. `CardSession`의 spell 렌더 코드는 방어용으로 남겨두되 정상 풀에선 도달하지 않음. (레벨 테스트 `buildQuiz`의 `type:"spell"`은 별개 컴포넌트라 그대로.)
- **단어카드 입력은 카드 내내 언마운트되지 않는 단일 "지속 입력"(`.ans-catch`, 보이지 않음)이 받고, 표시는 `answerDisplay`(셀/박스/정답 fill)가 담당(2026-07-17).** 채점 후(`res`)에도 input이 살아있고 readOnly가 아니라(포커스 유지) 모바일 키보드가 안 내려감 — 예전엔 정답마다 input이 언마운트돼 키보드가 툭 내려갔었음. Enter는 `res`면 다음 문제, 아니면 확인. 하단 액션바(`.qbottom`)는 `position:sticky; bottom:var(--kb)`로 키보드 위에 붙어 항상 눌림. 새 카드/다음으로 넘어갈 때 `focusHidden()`으로 포커스 유지(탭 제스처 안에서). readOnly를 쓰면 iOS가 키보드를 숨기므로 쓰지 말 것(대신 `res`일 때 onChange에서 무시).
- **글자수 힌트 셀 입력 프리즈 수정(2026-07-17)**: cell 입력은 onChange에서 `[^A-Za-z']` 필터+길이 컷을 하는데, 필터 결과가 현재 state와 같으면 리렌더가 안 나 iOS 컨트롤드 input의 DOM 값이 사용자가 친 그대로 남아 "지우고 다시 입력할 수 없는" 프리즈가 났음. `e.target.value = out`으로 DOM을 강제 동기화하고 `maxLength`를 제거(onChange가 항상 잘라줌)해 해결.
- **단어카드↔듣기 세션이 같은 단어를 연달아 안 내게(2026-07-17)**: `pickSession`이 `RECENT_KEYS`(인메모리, 최근 40개)에 낸 항목을 기록하고 다음 세션에선 뒤로 미룸 — 카드 하고 이어서 듣기를 켜면 서로 다른 단어가 나온다. 카드·듣기 둘 다 정답 시 `SRS.grade`로 복습을 갱신하므로 한 모드에서 낸 due를 다음 모드에서 빼도 복습 누락이 없다. 부족하면 그때만 최근 항목으로 채우고, 새로고침하면 초기화(리뷰를 영구히 굶기지 않도록).
- **"지금 복습할 카드" 0 방지**: SRS 데모 시드 중 8개(`knock#0` 등)는 `demo` 미표시 + `due` 과거로 둬서 실제 복습 대기 큐로 남긴다 — 대시보드가 0으로 안 비고 홈 "복습 8"과 일치. 나머지 데모는 여전히 `demo`라 `pickSession`엔 안 섞임. 데모 항목에 `at`(학습 날짜, 최근 ~90일 6개씩 클러스터)도 부여.
- **"내가 학습한 단어 > 날짜별"**: SRS `at`(마지막 학습 시각, `grade`에서 기록)로 그룹핑. `learnedMode`(date/status) 세그먼트 토글, 기본 date.
- **보관함 즐겨찾기/내 단어장 바로가기**: `dictViewJump`(App)→`Dict`의 `viewJump` effect로 허브 안 거치고 해당 뷰로 바로 진입.
- **WordRow 뱃지 2줄**: CEFR은 윗줄 단독, 뜻·상태(완료/복습)는 아랫줄에 같이(`.meta-line`).

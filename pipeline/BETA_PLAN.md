## A. 프론트엔드 완성 — 내가 코드로 혼자 할 수 있는 것 (백엔드 불필요)

우선순위 순. 백엔드/디자인/외부계정 의존 없는 것만.

1. **활동 로그 레이어 + 뱃지 로직 실데이터화** — 지금 `BADGE_PROGRESS` 하드코딩 더미. 로컬 활동 로그(듣기 세션 완료·회화 세션·복습 수·어원 열람 등) 신설 → 뱃지 진행값을 SRS·스트릭·이 로그에서 실계산. 소셜/서버 의존 카테고리(친구초대·자립회화·주간대결·백분위)만 더미 유지(마커 표시).
2. **홈 더미→실데이터** — 스트릭·오늘의 루틴 진행·학습 캘린더 잔디·레벨 3칸을 로컬 실데이터로. (레벨 3칸은 레벨테스트 저장값 연동.)
3. **커스텀 페르소나 생성** — 회화 탭에서 사용자가 페르소나 만들기(이름·성격·프롬프트·아바타색). localStorage 저장 + `talkFeedbackAndNext` 프롬프트에 반영. (PLAN 4.6 "커스텀 페르소나 생성" 미구현분.)
4. **온보딩** — 최초 진입 플로우(레벨테스트 연결 포함). PLAN 여러 곳 미착수.
5. **문법 태그 detectGrammar()** — 코드는 내가, 데이터(G01~G04)는 배치. 데이터 먼저 필요.
6. **문장 퀴즈**(빈칸/딕테이션/어순/순간작문) — PLAN 4.4 미착수.
7. **오답 노트, 즐겨찾기/회화 수정문 출제 소스** — PLAN 4.3.5.
8. **흘려듣기(4.11)** — 지난 대화 오디오 재생. 설계만.

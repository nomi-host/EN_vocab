# 트랙 A 3단계 — 한국어 뜻·예문 작성 에이전트 지침

"NoEnglish"(한→영 어휘 학습 PWA, /home/user/EN_vocab) 어휘 20,000 확장(`pipeline/VOCAB_20K_PLAN.md` 3장)
중 트랙 A 3단계 — 큐레이션에서 채택된 신규 표제어의 **한국어 뜻과 예문을 작성**한다. 이 단계가
전체 계획에서 가장 규모가 큰(~74배치) 실제 콘텐츠 작성 단계다.

## 입력
배정 파일: `/home/user/EN_vocab/pipeline/work_expansion/EXP_<NN>.json`
= 배열, 각 항목:
```
{
  "word": "attorney",
  "pos": ["noun"],           // WordNet에서 기계 추출한 품사(noun/verb/adjective/adverb), 근거일 뿐 최종 아님
  "cefr": "B1",              // SUBTLEX 구어빈도 랭크 기반 근사치(참고용, 확신 있으면 조정 가능)
  "en": "a professional person authorized to practice law",  // WordNet 영영 정의 — 근거 참고용
  "syn": ["lawyer"]          // WordNet 동의어 후보 — 근거 참고용
}
```
**주의**: `en`/`syn`/`pos`는 WordNet에서 기계적으로 뽑은 **가장 흔한 synset 하나**일 뿐이라, 가끔
그 단어의 실제 가장 흔한 뜻과 다를 수 있다(예: "ha"가 천문학 "시간각" 정의로 잘못 뽑히는 경우 —
실제로는 감탄사 "하"가 훨씬 흔함). **이 필드들을 그대로 베끼지 말고, 학습자가 실제로 마주칠 가장
흔한 뜻이 무엇인지 스스로 판단해서 작성할 것.** WordNet 필드가 명백히 부적절하면 무시하고 일반
영어 지식으로 작성하되, 그런 경우 `review_parts`에 사유를 남긴다.

## 작성 방식
1. **`ko`**: 한국어 뜻, 사전식 간결체(콤마로 2~3개까지). 그 단어의 **가장 흔하고 실용적인 뜻**을 우선.
   - **음차뿐인 뜻 지양**: `router→라우터`만 쓰지 말고 `라우터, 경로 지정 장치`처럼 네이티브 뜻 병기.
   - 비속어·민감 소재 단어(큐레이션에서 damn/hell급으로 통과된 것들)는 뜻을 순화해서 쓰지 말고
     정확하게 쓰되, 자극적인 톤은 피하고 사전적으로 담백하게.
2. **`ex`**: `{ "en": "<그 단어를 실제로 보여주는 자연스러운 문장>", "ko": "<자연스러운 한국어 번역>" }`.
   직역·딱딱한 번역 금지, 실제 대화·글에서 쓰일 법한 문장.
3. **다의어 여부**: 이 단계에서는 **다의어 분리를 하지 않는다**(flat `ko`+`ex` 하나만) — 뚜렷이
   다의어인 단어도 여기선 대표 뜻 하나만 쓰면 된다. 다의어 분리는 이후 별도 라운드(트랙 A 5단계,
   기존 `senses_parts/` 파이프라인)에서 이 표제어들이 자동으로 편입되어 처리된다.
4. **CEFR 재조정**: `cefr` 필드는 랭크 기반 근사치라 부정확할 수 있다. 명백히 안 맞으면(예: 매우
   일상적인 단어인데 C2로 잘못 매겨진 경우) 조정해서 출력에 반영해도 된다 — 조정한 경우만
   `review_parts`에 사유를 남겨라(안 바꾼 건 남기지 않아도 됨).

## 프로젝트 규칙 (CLAUDE.md)
- **이모지 절대 금지.**
- 뜻·예문 전부 자연스러운 한국어(번역투 금지).

## 출력 스키마 (표제어 하나당)
```
{ "word": "attorney", "ko": "변호사", "ex": { "en": "She hired an attorney to handle the case.", "ko": "그녀는 그 사건을 처리하려고 변호사를 고용했다." }, "cefr": "B1" }
```
`cefr`은 입력값을 그대로 쓰거나(대부분) 조정한 값(드물게)을 넣는다 — 반드시 포함할 것(merge 단계에서
이 값을 그대로 사용, 입력의 `cefr`을 누락하면 안 됨).

## 출력 파일 (정확히 이 둘만 작성)
1. `/home/user/EN_vocab/pipeline/batches/expansion_<NN>.json` — 배열, 배치의 **모든 단어** 포함(100% 커버).
2. `/home/user/EN_vocab/pipeline/review_parts_expansion/expansion_<NN>.json`
   — `{ "<소문자 word>": "<한국어 사유>", ... }`. WordNet 근거를 무시하고 작성한 경우·CEFR을
   조정한 경우·확신 낮은 경우만. 없으면 `{}`.

## 엄수
- `merge_cefrj.js` 실행·`words.js` 재생성·git 커밋/푸시 **금지**. 지정된 두 파일 외 어떤 파일도 수정 금지.
- 배치 단어 **100% 커버**. 하나도 건너뛰지 말 것. 유효한 JSON만.
- **중간 저장 필수(세션 리밋 대비)**: 25~30단어 단위로 끊어서 그때그때 두 출력 파일을 갱신
  (기존 내용에 누적, 유효 JSON 유지).

## 반환 (최종 메시지, 간결히)
- 처리 단어 수, CEFR 조정 건수, 검수 플래그 개수.
- WordNet 근거가 부적절해서 무시하고 작성한 재미있는 사례 2~3개.
- 가장 흥미로운 뜻·예문 2개 전문(품질 스팟체크용).

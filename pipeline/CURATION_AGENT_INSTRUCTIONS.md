# 트랙 A 큐레이션 에이전트 지침 (표제어 후보 채택/기각 판정)

"NoEnglish"(한→영 어휘 학습 PWA, /home/user/EN_vocab) 어휘 20,000 확장(`pipeline/VOCAB_20K_PLAN.md` 3장,
트랙 A 2단계)의 큐레이션 배치를 판정한다. **이 단계는 한국어 작성이 아니라 accept/reject 판정만 한다.**

## 배경
`pipeline/expansion/candidates.json`은 SUBTLEX-US 구어빈도 74,286어를 "소문자만·WordNet 레마 존재·
SEED 미보유·기보유 원형의 **규칙** 굴절형 제외"라는 **기계적** 필터로만 거른 것이라 노이즈가 많이
남아있다(예: 대명사·조동사 파편 s/t/m/don/re/d, 비속어 shit/fuck/bitch, 고유명사, 불규칙 굴절형인
sold/broke/chosen처럼 규칙 필터로 못 거른 것들, 영국식 철자 centre/labour 등). 이 배치는 그걸 사람
수준으로 다시 걸러 실제 학습 사전에 넣을 표제어만 추린다.

## 입력
배정 파일: `/home/user/EN_vocab/pipeline/expansion/curation_batches/CUR_<NN>.json`
= 배열 `{ "word": "<소문자>", "rank": <SUBTLEX 구어빈도 랭크, 작을수록 흔함> }`.

## 판정 기준
**기각 사유** (하나라도 해당하면 기각):
1. **기보유 단어의 파생·굴절형**(불규칙이라 기계 필터를 통과한 것): 예 sold(sell)·broke(break)·
   chosen(choose)·better/best(good) 등 — 현재 시드에 원형이 있는지 확인할 것(아래 검증 방법 참고).
   **주의**: 트랙 B로 이미 별도 처리 중인 "품사 추가"(예: field를 동사로)와는 다른 문제 —
   여기서 말하는 건 단순 활용형·비교급/최상급이 랭크 목록에 그대로 섞여 들어온 경우다.
2. **대명사·조동사·전치사 등의 구어체 파편/축약형 잔재**: s·t·m·don·re·d·isn·ain·em·y·th·la·ha·
   ma 같은 것(아포스트로피가 SUBTLEX 토큰화 과정에서 떨어져 나온 것) — 독립 표제어가 아님.
3. **고유명사**: 인명·지명·브랜드명(paris, twitter, michigan류) — 대문자 표기가 정석인 단어.
4. **비속어**: 학습앱에 부적합한 수위(성적 비속어·인종/혐오 비속어 등). 단, 일상 구어에서 정말 널리
   쓰이고 순화된 뜻으로도 통용되는 경계선 단어(예: damn, hell, crap 정도)는 채택 가능 — 판단 기준은
   "부모가 자녀 영어 학습앱에서 이 단어를 보고 불쾌해할 정도인가".
5. **영국식 철자 변형**: 이미 미국식 철자가 시드에 있으면 기각(centre/labour/theatre류).
6. **화석어·사어**: 현대 영어에서 거의 안 쓰이는 고어체.
7. **명백한 오타/비단어**: SUBTLEX 토큰화 잡음.

**채택**: 위에 해당 안 되고 실제 학습 가치가 있는 일반 어휘(뉴스어·구어·생활어 포함). 애매하면
**채택 쪽으로 판단**하되 `note`에 애매함을 남겨라(과잉 기각보다 과잉 채택이 안전 — 나중 단계에서
한국어 작성자가 한 번 더 거를 기회가 있다).

## 출력 스키마
배치의 **모든 단어**에 대해 판정 결과를 남긴다(채택/기각 모두 포함 — 감사 추적용):
```
{ "word": "<소문자>", "rank": <입력값 그대로>, "accept": true|false, "reason": "<기각 시 위 1~7 중 어느 사유인지 한국어로 간단히, 채택 시 빈 문자열 또는 애매했던 경우만 note>" }
```

## 출력 파일 (정확히 이 파일 하나만 작성)

— 위 스키마 객체의 배열, 입력과 같은 순서·같은 개수(100% 커버).

## 엄수
- **한국어 뜻·예문은 이 단계에서 작성하지 않는다** — 그건 다음 단계(3단계, 별도 배치)의 몫.
- CEFR 태깅도 이 단계에서 하지 않는다(랭크 기반으로 기계적으로 매길 것이라 불필요).
- 다른 어떤 파일도 수정 금지. merge_cefrj.js 실행·words.js 재생성·git 커밋 금지.
- 배치 단어 100% 판정. 유효한 JSON만.
- **중간 저장 필수**: 100~150단어 단위로 끊어서 출력 파일 갱신(세션 리밋 대비).

## 반환 (최종 메시지, 간결히)
- 판정 단어 수, 채택/기각 개수와 채택률(%).
- 기각 사유별 개수 분포(1~7 카테고리).
- 애매해서 채택 쪽으로 넘긴 경계 사례 3~5개(단어+이유).

# 트랙 B(기존 단어 sense 갭) 작성 에이전트 지침

"NoEnglish"(한→영 어휘 학습 PWA, /home/user/EN_vocab)의 어휘 20,000 확장(`pipeline/VOCAB_20K_PLAN.md`)
중 트랙 B — **이미 시드에 있는 단어에, 코퍼스 실측상 실제로 쓰이는데 빠진 품사의 의미를 추가**한다.
(예: abandon은 동사 뜻만 있는데 명사 "wild abandon"(거리낌 없음) 뜻이 빠져 있음.)

## ★가장 중요 — 기존 뜻을 절대 잃어버리지 말 것
이 트랙은 일반 다의어(senses) 배치와 다르다. 대상 단어는 **이미 뜻이 있는 상태**이고, 거기에
**새 품사의 뜻을 추가**하는 것뿐이다. 최종 산출물은 항상 "기존 뜻 전부 + 새 뜻"을 합친 **완전한
배열**이어야 한다 — 새 뜻만 쓰면 병합 스크립트가 그걸로 통째로 덮어써서 기존 뜻이 사라진다.

## 입력
배정 파일: `/home/user/EN_vocab/pipeline/work_trackb/TRACKB_<NN>.json`
= 배열, 각 항목:
```
{
  "word": "abandon",
  "cefr": "B1",                    // 표제어 전체의 기존 CEFR(참고용, 새 sense의 cefr은 아래서 별도 판단)
  "missingPosKo": ["명사"],         // 추가해야 할 품사(한글) — 이것만 새로 씀, 있는 품사는 건드리지 않음
  "wn": [{ "pos":"명사", "def":"...", "syn":[...] }],  // WordNet 근거(missingPosKo에 해당하는 synset만).
                                     // 없을 수도 있음(빈 배열) — 그래도 실제 갭이 확인된 단어이니
                                     // 일반 영어 지식으로 작성하고 review_parts에 사유를 남길 것.
  "existing": [ {sense 스키마 객체...} ],   // 이미 senses[]가 있는 경우 — 이 배열을 그대로 보존해서
                                              // 최종 출력에 다시 포함시킬 것 (절대 삭제/수정 금지)
  // 또는 (existing이 없는 경우, 즉 이 단어가 아직 flat ko/en/ex만 있던 경우):
  "existingFlat": { "pos":["verb"], "ko":"버리다, 포기하다", "en":"...", "ex":{"en":"...","ko":"..."} }
}
```
`existing`과 `existingFlat`은 항상 둘 중 하나만 있다.

## 방식
1. **`existing`이 있는 경우**: 그 배열을 그대로(수정 없이) 복사해오고, `missingPosKo`에 해당하는
   새 sense(1개, 드물게 2개 이상도 가능 — `wn` 후보가 뚜렷이 갈리면)를 만들어 배열 끝에 추가.
   `lv`는 기존 sense들의 최대값+1로(새 품사는 대개 덜 흔한 뜻이라 lv도 뒤로).
2. **`existingFlat`만 있는 경우**: 먼저 그 flat 정보를 **sense 스키마 객체로 변환**해서 1번째 sense로
   만들고(`cue`는 간단히 만들어도 됨, `lv:1`, `cefr`은 표제어의 기존 cefr 그대로, `ko`/`en`/`ex`는
   flat 값 그대로 이관 — 절대 새로 창작하지 말 것), 그 뒤에 새 sense(`lv:2`)를 추가.
3. 새 sense 작성 방식은 기존 다의어 작업(`AGENT_INSTRUCTIONS.md`)과 동일한 하이브리드 원칙:
   `wn`을 근거로 삼되 과분할하지 말고, 학습자가 실제로 쓸 뜻 1개(드물게 2개)만.
4. `wn`이 빈 배열이면 일반 영어 지식으로 작성 가능하나, 반드시 `review_parts`에 사유 기록
   (`"WordNet 후보 없음 — 일반 지식으로 작성"`).

## 출력 스키마 — 새로 추가하는 sense 하나당 (기존 다의어 작업과 동일, 참고: `senses_parts/00_pilot.json`)
```
{
  "cue": "<짧은 한국어 맥락 큐>",
  "ko": "<한국어 뜻, 사전식 간결체>",
  "pos": "<'명사'|'동사'|'형용사'|'부사' 등>",
  "lv": <정수 — 기존 sense들 다음 순번>,
  "cefr": "<'A1'~'C2' — 이 의미 자체의 난이도>",
  "en": "<학습자용 쉬운 영영 정의 한 문장>",
  "syn": ["<이 의미 문맥에 한정된 치환어, 없으면 []>"],
  "ex": [{ "en": "<자연스러운 예문>", "ko": "<자연스러운 번역>" }]
}
```

## 프로젝트 규칙 (CLAUDE.md)
- 한국어: 뜻·큐는 사전식 간결체. **이모지 절대 금지**.
- 예문은 그 의미를 실제로 보여주는 자연스러운 문장 + 자연스러운 번역(직역 금지).
- 음차뿐인 뜻 지양 — 네이티브 뜻 병기.

## 출력 파일 (정확히 이 둘만 작성)
1. `/home/user/EN_vocab/pipeline/senses_parts/TRACKB_<NN>.json`
   — `{ "<소문자 word>": [ {완성된 senses 배열 — 기존+신규 전부}, ... ] }`. 배치의 **모든 단어** 포함.
2. `/home/user/EN_vocab/pipeline/review_parts/TRACKB_<NN>.json`
   — `{ "<소문자 word>": "<한국어 사유>", ... }`. 확신 낮은 단어만(WordNet 후보 없음·의미 경계 모호 등).
   없으면 `{}`.

## 엄수
- merge_cefrj.js 실행·words.js 재생성·커밋/푸시 **금지**. 지정 두 파일 외 어떤 것도 수정 금지.
- 배치 단어 **100% 커버**. 하나도 건너뛰지 말 것. 유효한 JSON만.
- **기존 senses/flat 정보를 축약·삭제·재작성하지 말 것** — 그대로 보존 + 새 sense만 추가.
- **중간 저장 필수(세션 리밋 대비)**: 25~30단어 단위로 끊어서 그때그때 두 출력 파일을 갱신(누적, 유효 JSON 유지).

## 반환 (최종 메시지, 간결히)
- 처리 단어 수, `existing`/`existingFlat` 분포, 검수 플래그 개수.
- 가장 흥미로운 sense 추가 2개 전문(품질 스팟체크용).

# 문법 태그 설명문 작성 에이전트 지침

"NoEnglish"(한→영 어휘 학습 PWA, /Users/minyoungkim/EN_vocab)의 문장 단위 문법 태그 마스터 데이터를 작성한다.
배경·설계 전체는 `docs/PLAN.md` 2.5절 참고. 다의어(senses) 배치와는 별개 트랙 — 이건 표제어별 데이터가 아니라
"관계대명사 계속적 용법" 같은 문법 포인트당 **고정 설명문 1개**를 쓰는 작업이다(전체 목표 79개, 한 번 쓰면 거의 안 바뀜).

## 입력
배정 파일: `/Users/minyoungkim/EN_vocab/pipeline/work_grammar/<BATCH>.json`
= 배열 `{ "id", "category", "name_kr" }`(빈 스텁 — 이 셋만 미리 정해져 있고 설명문·예문은 아직 없음).

## 방식
먼저 `/Users/minyoungkim/EN_vocab/pipeline/grammar_tags_parts/00_pilot.json`을 읽고 스타일을 정확히 맞춘다
(`relative-clause-continuous`/`passive-simple`/`subjunctive-past` 3개가 파일럿).

각 항목에 대해:
1. **설명문(`desc_kr`)**: 이 문법 포인트가 정확히 무엇이고 언제/왜 쓰는지, 한국 학습자가 학교 문법 시간에 배운
   용어를 그대로 살려 3~5문장으로 설명. "규칙 나열"이 아니라 "왜 이런 형태가 되는지"가 감이 오게 쓸 것
   (파일럿의 `subjunctive-past`처럼 "형태는 과거지만 뜻은 현재"처럼 헷갈리는 포인트를 명시).
2. **예문(`example`)**: 그 문법 포인트가 뚜렷하게 드러나는 자연스러운 문장 1개 + 자연스러운 한국어 번역
   (직역·딱딱한 투 금지 — 다의어 배치와 동일 원칙).
3. 같은 대분류(`category`) 안에서 다른 항목과 설명이 겹치지 않게 — 예를 들어 "수동태 단순"과
   "수동태 진행"을 같은 말로 설명하지 말고 그 둘의 차이가 설명문만 보고도 구분되게 쓸 것.

## 출력 스키마 (항목 하나당) — 파일럿과 정확히 일치시킬 것
```
{
  "<id>": {
    "category": "<입력의 category 그대로>",
    "name_kr": "<입력의 name_kr 그대로>",
    "desc_kr": "<3~5문장, 학습자용 설명>",
    "example": { "en": "<그 포인트가 드러나는 자연스러운 예문>", "ko": "<자연스러운 한국어 번역>" }
  },
  ...
}
```

## 프로젝트 규칙 (CLAUDE.md)
- **이모지 절대 금지**(🎉⚠️ 등 어디에도).
- 예문·설명문 모두 자연스러운 문장/번역(직역·딱딱한 투 금지).

## 출력 파일 (정확히 이 하나만 작성, 그 외 어떤 파일도 건드리지 말 것)
`/Users/minyoungkim/EN_vocab/pipeline/grammar_tags_parts/<BATCH>.json` — `{ "<id>": {항목}, ... }`.
배치의 **모든 id** 포함. 확신이 낮은 항목(예: 카테고리 경계가 애매한 문법 포인트)이 있으면 `desc_kr` 끝에
괄호로 `(검수 필요: 이유)`를 짧게 덧붙이는 정도로만 표시 — 다의어처럼 별도 review 파일은 만들지 않는다
(이 트랙은 배치당 항목 수가 적어 파일 하나로 충분).

## 엄수
- `words.js`/`index.html`/`lib/compromise.min.js` 등 이 작업과 무관한 어떤 파일도 수정 금지.
- merge 스크립트(아직 없음)를 새로 만들거나 실행하지 말 것 — grammar_tags.json으로의 최종 병합은 이 작업 범위 밖.
- 커밋/푸시 금지.
- 배치 항목 **100% 커버**. 하나도 건너뛰지 말 것.
- 유효한 JSON만(파싱 확인).
- **중간 저장 필수(세션 리밋 대비)**: 배치가 20개 안팎으로 다의어보다 훨씬 작지만, 그래도 **10개 단위로 끊어서
  그때그때 `grammar_tags_parts/<BATCH>.json`을 갱신(기존 내용에 누적, 유효한 JSON 유지)**할 것. 한 번에 몰아서
  마지막에만 쓰면 중간에 세션 리밋에 걸렸을 때 전체가 유실된다(실제로 다의어 배치에서 이 문제로 한 배치를
  통째로 날린 적 있음, 2026-07-18).

## 반환 (최종 메시지, 간결히)
- 처리 항목 수.
- 검수 필요 표시한 항목 수.
- 가장 설명하기 까다로웠던 항목 1~2개의 전문(품질 스팟체크용).

NoEnglish 히어로 영역 가이드

홈 화면 상단 히어로 섹션(캐릭터+노트 배경+스티커)의 현재 구현 스펙 — index.html의 실제 코드 기준 (2026-07-20 최신화). 이 문서는 거동/치수 레퍼런스용이고, 캐릭터 아트 자체의 제작 파이프라인은 design/hero_cat_lottie/README.md(Lottie 제작 과정)와 stage-pixel-spec.svg(char2~4 정지 포즈 배치 블루프린트)가 정본입니다. 값을 바꿔도 이 페이지엔 반영 안 됨(참고용) — 실제로 바뀌면 index.html의 HeroScene을 고치고 이 문서도 같이 갱신할 것.

전체 구조 & 레이어 순서

히어로 박스는 .hero-scene — width:100%; aspect-ratio:350/220, 배경 #FAF3E5 + var(--card-border) 1px 외곽선(노트 줄공책 느낌은 .ruled-paper가 별도 담당). position:relative; z-index:0으로 독립 스태킹 컨텍스트를 만들어서, 그 안의 자식 z-index는 전체 화면 모달(.dlg-backdrop 등, z-index:60)과 절대 충돌하지 않음 — 서로 다른 스태킹 컨텍스트라 숫자만 보고 비교하면 안 됨.

streakStage(days){ return days<3?1:days<7?2:days<14?3:4 } — 3/7/14일 경계, 0단계 구분 없음(2026-07-19 사용자가 명시적으로 이 값으로 확정, 다른 경계값 재제안은 반려됨). 3·4단계는 전용 아트가 아직 없어 둘 다 "날기" 애니메이션을 공유한다.

캐릭터 박스 위치/크기 (스테이지별 다름 — 실측 좌표)

함정: Lottie comp(정사각 200×200)는 캐릭터 실루엣 자체보다 훨씬 크게(내부 여백 포함) 렌더되므로, "이 %만큼 크게 보이면 좋겠다"는 목표 실루엣 비율을 박스 치수에 그대로 넣으면 안 된다. 실제 렌더를 픽셀 단위로 실측(연결요소 분석)한 뒤 comp 여백을 역산해서 박스 %를 정해야 함 — 자세한 계산 과정은 hero_cat_lottie/README.md "박스 재계산" 절 참고. 포즈(걷기/달리기/날기)마다 실루엣 비율이 달라 박스 값도 전부 다름.

캐릭터 탭 안내 팝업 (2026-07-20 신설)

캐릭터(.hero-cat-hit)를 탭하면 히어로 영역만(화면 전체 아님) 살짝 딤되고 안내 카드가 뜬다: "연속학습을 이어가서 다른 포즈의 캐릭터를 만나보세요!" 배경(딤 영역) 탭하면 닫힘, 카드 자체를 탭하면 안 닫힘(이벤트 버블링 차단).

구현 메모: .hero-cat 자체는 pointer-events:none이라(스티커·바닥과의 인터랙션을 방해하지 않기 위한 기존 설계) 직접 탭 핸들러를 못 단다 — 안쪽에 pointer-events:auto인 .hero-cat-hit 래퍼를 하나 더 감싸 그 안에서만 탭을 받는다. 새로운 히어로 하위 인터랙션을 추가할 때도 이 패턴(부모는 pointer-events:none 유지, 실제로 반응해야 할 자식만 auto로 재활성화)을 따를 것.

스티커 레이어

기본 6종(스마일·KEEP IT UP·Yes English 말풍선·Well begun 마테·클로버·꽃) — 정적 이미지 + CSS 부유(floaty, prefers-reduced-motion이면 정지) + 소프트 그림자(베이크됨). 스마일만 눈썹 좌우 다른 타이밍으로 들썩이는 개별 Lottie.
동작구현
이동꾹 눌러(long-press 320ms) 드래그. 히어로 밖 최대 50px까지만 — overflow:hidden 마스크로 그 밖은 안 보임. 위치는 localStorage.
겹침 순서고정 z-index 대신 order 배열(집으면 맨 앞으로) — localStorage 저장. 캐릭터(z-index:100)보다는 항상 아래.
삭제("치움")같은 롱프레스로 우상단 X 배지 노출 → 탭하면 "치움" 처리(완전 삭제 아님, en_hero_sticker_removed_v1에 보관 — 스티커 꾸미기 화면에서 다시 놓을 수 있음).
선택 해제히어로 배경(빈 곳) 탭.

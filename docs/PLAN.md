## 1. 개요

- **목표**: JP vocab(ピッ!とVOCA) 앱의 검증된 구조를 복제·확장한 **개인용 영어 학습 웹앱(PWA)**.
- **핵심 축 3가지**: ① 망각곡선 기반 단어 암기 ② 퀴즈 ③ 회화력(작문 첨삭 + 페르소나 회화).
- **아키텍처**: JP 앱과 동일.
  - 단일 `index.html` (React CDN, 빌드 없음) + `words.js` 등 시드 데이터 + PWA(manifest).
  - Vercel 서버리스 프록시 `api/proxy.js` (Gemini + Google TTS 키 은닉, CORS/레이트리밋).
  - 저장: `localStorage`(학습 상태) + `IndexedDB`(TTS 오디오·단어정보 영구 캐시 → API 호출 최소화).

### 2.2 판단 근거

- **단어정보**: 뜻·발음·유의어·반의어까지는 무료 API 조합(Free Dictionary + Datamuse)으로 충분. 단, 실시간 API 의존 대신 JP 앱처럼 **시드 데이터에 미리 굽고(words.js/synonyms.js), 없는 것만 런타임 API + IndexedDB 캐시**가 안정적.
- **한국어 뜻**: 네이버/다음은 공개 API를 제공하지 않고 크롤링은 약관 위반 소지 → 대신 **오픈 사전 3종 병합(2.3 참조)**으로 대부분 커버하고, 빈 곳만 LLM 번역. 학습 중 발견되는 오역은 JP 앱의 오류 신고 흐름으로 수정.
- **어원/어근**: "같은 어원 공유 단어 타고타고"는 etymology-db에서 라틴/그리스 어근 기준으로 역인덱스를 만들어 시드화. 어근·접사 카드(com-, inter-, -tion 등)는 양이 작아 자체 데이터로 관리하는 편이 품질이 좋음.
- **문법정보**: "긁어올" 무료 소스가 없는 영역이지만 LLM도 불필요 — **브라우저 내 POS 파서(MIT 라이선스) + 규칙 매칭으로 태그를 자동 검출**하고, 태그별 설명문(관계대명사, 가정법, to부정사… ~100개)만 1회 자체 작성. 규칙이 못 잡는 예외만 LLM 폴백.
- **외부사전 연결**: URL 스킴으로 무료 연동 — 네이버 영어사전(`en.dict.naver.com/#/search?query=단어`), Longman, Cambridge, Etymonline(어원 확인용).

### 2.3 LLM 최소화 파이프라인 (생성 → 번역으로 격하)

토큰 비용과 QA 부담을 줄이기 위해 **LLM의 역할을 "콘텐츠 생성"에서 "번역·갭 필링"으로 축소**한다.

1. **표제어·레벨**: CEFR-J Wordlist(A1–B2) + Octanove Vocabulary Profile(C1–C2) — 무LLM
2. **뜻(한국어)**: 오픈 사전 3종(한국어판 위키낱말사전, 영어판 번역 섹션, 국립국어원 한영 학습사전 역인덱스) 병합 → 커버리지 측정 → **미커버 단어만 LLM 번역** (상위 5,000 단어는 오픈 사전 커버리지가 가장 높은 구간이라 갭이 작음)
3. **예문**: Tatoeba 영어 코퍼스에서 표제어별로 길이·난이도 필터로 선별(무LLM) → 영한 페어가 이미 있으면 그대로, 없으면 **LLM은 한국어 번역만**
4. **유의어·반의어·품사 파생·어원**: WordNet + etymology-db에서 기계 추출 — 무LLM
5. **문법 태그**: 브라우저 내 POS 파서 + 규칙 매칭 — 무LLM. 설명문 ~100개만 1회 작성
6. **검수(QA)**: 생성물 검수가 아니라 **번역 검수만** 남음. 번역은 원문 대조가 가능해 검수 난이도가 낮고, Tatoeba 인간 번역·사전 출처 데이터는 검수 자체가 불필요

예상 배치 비용: 예문 번역 ~1만 문장 + 뜻 갭 필링 → **1회 $2 미만**. 런타임 LLM 사용처는 학습 데이터가 아닌 기능(첨삭·회화·AI 질문)으로 한정된다.

라이선스 주의: 위키낱말사전·국립국어원 사전 파생 데이터는 CC BY-SA(동일조건) — **데이터 파일(words.js 등)에 출처 고지 주석을 넣고 해당 파일은 CC BY-SA 유지**(코드와 분리). 공공누리 1유형은 출처 표시만으로 상업 이용까지 허용.

### 2.4 다의어 원칙 — 학습 단위는 단어가 아니라 "의미(sense)"

영어는 한 표제어에 뜻이 여럿(save = 구하다/아끼다/저장하다/모으다…)이므로, **의미별로 별도 항목으로 분리해 별도 예문을 붙이고 SRS도 의미 단위로 돈다** (선행 어휘앱(국내 대표)와 동일 원칙).

- **스키마**: 표제어에 `senses[]` — 의미마다 `{cue(맥락 큐: "위험으로부터"), ko, en(영영), pos, lv(의미 빈도 레벨), ex[]}`. 단일 의미 단어는 기존 flat 필드 유지.
- **학습 단위**: 카드·SRS 박스·산출 큐 모두 `단어#의미` 단위. "학습에 추가"도 의미별 버튼.
- **출제 규칙**: 같은 단어의 다른 의미는 **같은 세션에 함께 출제하지 않음** — 유사 항목 동시 학습은 간섭(interference)으로 기억을 방해한다는 어휘습득 연구 원칙. 의미 간 간격을 두고 순차 도입(빈도 높은 의미부터).
- **파이프라인**: WordNet synset이 의미 분리를 기계적으로 제공하나 과분할이 심함 → **의미 빈도 상위 N개로 병합·선별**(단어 빈도-의미 분포 기준) 후 한국어 뜻·맥락 큐는 오픈 사전 3종 + LLM 갭필. 맥락 큐는 짧은 한국어 부사구로 통일("위험으로부터", "나중에 쓰려고").
- **표시**: 사전 행 = "구하다 외 5개 뜻" + `뜻 6` 뱃지. 상세 = 의미 카드 리스트(첫 의미 펼침, 탭 토글).

## 3. API 비용 · 스케일 분석 (유료 스케일인가?)

**결론: 개인 사용 기준 월 0 ~ 3달러. 유료 스케일 아님. 웹앱으로 전부 가능.**

| 용도 | 산정 | 월 비용 |
|---|---|---|
| 회화 (Gemini 2.5 Flash, $0.30/1M 입력·$2.50/1M 출력) | 하루 40턴 × (입력 2K + 출력 0.3K 토큰) ≈ 월 280만 입력/36만 출력 토큰 | **≈ $1.7** |
| 작문 첨삭 | 하루 20문장 × (0.5K+0.3K) | ≈ $0.1 |
| 단어정보/예문 생성 | 수동 추가 시만, 캐시됨 | ≈ $0 |
| TTS (Neural2) | 회화+카드 하루 약 8천 자 → 월 24만 자, **무료 티어 100만 자 이내** + IndexedDB 캐시 | **$0** |
| STT | Web Speech(무료) 기본, iOS PWA 폴백 Gemini 오디오 하루 20분 | ≈ $1 이하 |
| 발음 진단 (Azure Pronunciation Assessment) | 표준 STT 요금(시간당 약 $1), 하루 10분 | ≈ $5 이하 |
| 사전류 (Free Dictionary/Datamuse/MW) | 전부 무료 티어 | $0 |

- Gemini는 **무료 티어(Flash 계열 하루 1,500 요청)**만으로도 개인 사용량을 대부분 커버 → 사실상 $0 운용도 가능.
- 스케일 리스크는 "여러 사용자 공개 배포" 시에만 발생. 개인+지인 수준이면 JP 앱의 프록시 레이트리밋 구조로 충분.

## 4. 기능 명세

### 4.1 단어 학습 (JP vocab 구조 복제 + 확장)

JP 앱에서 그대로 복제: Leitner 망각곡선 SRS(하루 신규 N개 + 자동 복습), 단어/예문 TTS, 흘려듣기, 잔디(스트릭 캘린더), 통계&설정, 오류 신고, 업데이트 안내(WHATS_NEW), 카테고리(JLPT → **CEFR A1~C1 + 카테고리별 단어장(비즈니스/여행 등) + 내 단어**), 온보딩 레벨별 신규 학습 비율.

**수동 단어 추가 (차별점)**: 선행 어휘앱(국내 대표)류는 자사 DB에 있는 단어만 저장 가능하지만, 이 앱은 JP 앱처럼 **DB에 없는 단어도 자유롭게 추가** — 오픈 사전(시드+Free Dictionary)에서 자동 조회하고, 없으면 LLM이 뜻·예문을 즉석 생성(사용자 트리거의 소량 호출이라 2.3 LLM 최소화 원칙과 충돌 없음). 추가된 단어도 SRS·퀴즈·타고타고에 완전 편입.

**진입점 = 사전 탭(5.3)**: 검색창에 입력 → DB에 있으면 단어 상세 팝업, 없으면 "이 단어 추가하기" 버튼 노출(회화·작문 중 모르는 단어를 만나도 말풍선 단어 탭 → 같은 흐름). 사전 탭의 목록 뷰: **전체**(CEFR/주제 필터) · **학습 중** · **학습 완료** · **내가 추가한 단어** — 각 항목은 SRS 상태(박스 단계·다음 복습일) 뱃지 표시.

**단어 상세 화면 확장** (타고타고 공부의 핵심):

- 뜻(한/영), 발음기호, 품사, CEFR 레벨, 유의어/반의어(탭하면 해당 단어 상세로 이동)
- **예문에 사용된 단어**: 예문 내 단어 칩 → 탭하면 그 단어 상세로
- **연관 단어**
  - 같은 어원을 공유하는 단어 (spect → inspect, respect, spectacle…)
  - 같은 단어의 다른 품사 (decide → decision, decisive, decidedly)
- **어근/접사 칩**: `inter-` `com-` `-tion` 등 → 탭하면 그 어근을 쓰는 단어 목록 화면으로 (목록에서 또 타고 들어가기)
- 외부사전 링크(네이버/Longman/Cambridge/Etymonline), 즐겨찾기, 오류 신고

### 4.2 학습 카드 (레퍼런스 이미지 방식)

선행 어휘앱(국내 대표)식 "예문 빈칸 입력" 카드:

1. 상단: 상황 설명(한국어) + 한국어 문장(타깃 부분 강조)
2. 하단: 영어 예문에서 타깃 단어(구)만 빈칸 → **키보드 타이핑 또는 음성 모드로 입력**
3. 정답 시: 빈칸 채워지며 강조, 다음 복습 시점 표시("6분 뒤 복습")
4. 정답 후 액션 바: 다른 예문 / 오답 노트 / 말해보기(발음 따라하기) / 단어 정보 / AI 질문 답변
5. 하단 버튼: 느리게 듣기(x0.8) / 다시 듣기 / 다음 문제
6. 즐겨찾기(하트), 길이 보기(글자 수 힌트)

### 4.3 레벨 테스트 — 어휘력 + 문장구사력 이원 측정

**Part A. 어휘** (기존 설계)
- CEFR 밴드별 적응형 체크(가짜 단어로 허수 방지) + **인지(뜻 알기) / 산출(한국어 보고 영어 말하기) 이원 측정**
- 결과: 추정 어휘량, 표준어휘 커버리지 %, 상위 %

**Part B. 문장구사력** (어휘만으로는 "책으로 배운 사람"의 실제 회화력을 못 잡는다 — 산출 능력을 직접 측정)

1. **문장 따라 말하기 (Elicited Imitation)** — 문장을 듣고 그대로 재현(음성). 문장이 작업기억 용량을 넘으면 **내재화된 문법 없이는 재현 불가**하므로, SLA 연구에서 검증된 암묵적 문법 지식 측정법. 길이·복잡도 점증 8문항, STT 대조로 자동 채점.
2. **한→영 순간영작** — 한국어 단문을 30초 안에 영어로(음성/타이핑) 4~6문항. LLM 루브릭 채점: 어순 / 시제·상 / 관사 / 자연스러움 4축.
3. **어순 배열 스피드** — 청크를 순서대로 배열, **반응시간 기록** 5문항. SOV→SVO 재배열 자동화 수준을 계량.

**결과 통합**
- "어휘는 B1, **문장 구사는 A2**" 식으로 격차를 명시 — 타깃 사용자의 전형 프로필을 첫 화면에서 보여주는 것 자체가 동기부여이자 진단.
- 구사력 프로필(정확도·자동화 속도·복잡도)에 따라 **루틴 배합 자동 조정**: 구사력이 낮으면 어순 드릴·순간영작·청크 카드 비중 증가.
- 이후 학습 이력(SRS 정답률·회화 채점)으로 두 축 모두 지속 갱신 → 통계 그래프.
- 온보딩 시 1회 실시 → 신규 학습 단어 레벨 비율 결정(JP 앱 LEVEL_RATIOS 방식). 어휘 파트만 먼저 하고 구사력 파트는 건너뛰기 허용(진입 장벽 완화), 미실시 시 첫 주 학습 데이터로 추정.

### 4.3.5 듣기 세션 (딕테이션 — 루틴 2단계)

선행 어휘앱(국내 대표)의 듣기 훈련을 독립 세션으로 채택. **문장을 듣고 그대로 타이핑**하는 것이 기본 흐름.

1. 오디오 재생(문장 전체) → 타이핑 입력(단어 수만큼 빈칸) → 채점 → 정답·해석·다시 듣기
2. 보조 컨트롤: x0.8 느리게 · 다시 듣기 · **또박또박/자연 속도(연음) 토글**(4.12 연계) · 못 잡은 단어만 부분 재생
3. 출제 소스: 학습한 단어의 예문 우선(SRS 연동 — 듣기에서 맞히면 해당 단어 복습 가점), 즐겨찾기 문장, 회화 수정문
4. 하루 기본 10문장(루틴), 난이도는 문장 길이·속도로 조절
5. 오답은 오답 노트 축적 → 퀴즈·흘려듣기와 공유

### 4.4 퀴즈 (보조 훈련 — 루틴 외 선택)

- **단어 퀴즈**: 학습한 단어 대상 ① 뜻 맞추기(4지선다) ② 스펠링 맞추기(뜻+예문 듣고 타이핑)
- **문장 퀴즈**: 학습한 예문/즐겨찾기 문장 대상. 제안 방식(효과 순):
  1. **빈칸 채우기(cloze)** — 기본. 타깃 단어+콜로케이션 단위로 구멍
  2. **딕테이션** — 듣고 받아쓰기(전체 또는 부분), 청취력 동시 훈련
  3. **어순 배열** — 문장을 청크로 쪼개 순서 맞추기(어순 감각)
  4. **한→영 순간작문** — 한국어 문장 보고 음성으로 말하기, LLM 채점 (회화 연계, 상급)
  - 기본은 1+2 혼합 출제, 3·4는 설정에서 켜는 방식 제안
- 하루 10문제 이상 → 잔디 반영(JP 앱 규칙 복제)

### 4.5 작문 첨삭 (회화력 축 1)

- **카테고리 선택**: 여행, 직장, 옷가게, 여행지 스몰토크, 공항, 입국심사, 병원, 마트, 친구, 비즈니스 이메일 등
- 흐름: 한국어 단문 제시 → 사용자가 영어로 작문(타이핑/음성) → LLM 첨삭
- 첨삭 결과: **자연스러움 %**, 교정문(diff 강조), **더 자연스러운(idiomatic) 표현 1~2개**, 한 줄 설명
- **문장 즐겨찾기** → 단어장(문장 탭)에 저장
- 내 작문 이력/카테고리별 평균 자연스러움 통계

### 4.6 회화 (페르소나 챗봇, 회화력 축 2)

페르소나 목록(이름 + 속성 뱃지 + 2D 아바타 — 아바타는 사용자가 별도 생성해 이미지로 추가):

| 페르소나 | 속성 | TTS 보이스 매핑 |
|---|---|---|
| IT회사 직장동료 (픽코마 다니는 개발자) | 미국인 | en-US 남성 |
| 3년 사귄 애인 | 영국인 | en-GB |
| 여행지에서 처음 만난 사람 | 호주인 | en-AU |
| 마트 점원 | 미국, AAVE 억양(말투는 프롬프트로) | en-US |
| 공항 지상직 직원 | 불친절함 | en-GB 여성 |
| 의사 | 중장년 남성, 말이 빠름 | en-US 남성 + rate 1.2 |
| 카페 직원 | 20대 여성, 말이 빠름 | en-US 여성 + rate 1.2 |
| 지나가는 행인 | 도움 요청 상황 | en-US |
| 친절한 영어 튜터 | 30대, 뉴욕 발음 | en-US 여성 |
| 10년지기 친구 | 슬랭 다수 | en-US 남성 |

대화 흐름:

1. 페르소나 탭 → 프로필(이름/속성/시나리오) → **대화 시작하기**
2. **상대가 먼저 말함** — TTS 재생과 동시에 말풍선 표시
3. 토글 2개: **대사 표시 on/off**(리스닝 훈련), **번역 표시 on/off**
4. 내 응답: 음성(기본) 또는 타이핑 → 말풍선에 표시
5. 내 응답에 대한 **실시간 피드백 카드**(문법/자연스러움/대안 표현) → **OK를 눌러야 다음 질문 진행**
   - **피드백 2모드** (LINGUISTICS.md G7): ① 정확성 모드 = 현행 즉시 게이트 ② **유창성 모드 = 대화 중 무중단, 종료 후 일괄 리뷰**. 매 발화 교정은 수능 세대의 "정확성 모니터 과잉"을 강화할 위험이 있어, 주 1회 이상 유창성 모드를 권장 루틴에 포함
6. 대화 중 아무 문장이나 **즐겨찾기** 가능 → 단어장(문장 탭)에 저장
7. 대화 종료 시 요약(잘한 표현/교정 목록) — 오답 노트 연계

**사용자 추가 페르소나**: 사용자가 직접 페르소나 생성 가능 — 이름·관계·성격·억양(지역 보이스 선택)·상황을 정형 필드로 입력.
- **월 1명만 생성 가능**, 슬롯 교체는 기존 페르소나 삭제 후(대화 이력은 단어장 즐겨찾기만 보존).
- 프로필 이미지는 기본 캐릭터 세트(5.2)에서 선택.
- 입력값은 정형 필드로만 받아 시스템 프롬프트에 주입(자유 텍스트 프롬프트 인젝션 방지).

### 4.7 단어장 (즐겨찾기 허브)

- 탭 2개: **단어** / **문장**
- 단어 탭: 즐겨찾기 단어 → 단어 상세로 타고타고 진입
- 문장 탭: 작문·회화·카드에서 저장한 문장. 상세에서:
  - 문장 TTS, 번역, 출처(회화: 어느 페르소나와의 대화인지)
  - **사용된 단어 칩**(→ 단어 상세), **사용된 문법 태그**(→ 문법 설명 + 같은 문법의 다른 저장 문장)

### 4.8 발음 진단

- 흐름: 작문 첨삭·회화에서 **수정문을 받으면 → 따라 말하기 → 정확도 진단**. 학습 카드의 "말해보기"도 같은 엔진 사용.
- 엔진: **Azure Speech Pronunciation Assessment** — 음소/단어/문장 단위 정확도(accuracy) + **프로소디(강세·억양·속도·리듬)** + 유창성 점수를 반환. 발음 평가의 사실상 업계 표준(언어학습 앱 다수 사용).
  - 비용: 표준 STT 요금과 동일 수준(시간당 약 $1) → 하루 10분 진단 시 **월 $5 미만**. 개인용 무리 없음.
- 화면: 문장 위에 **단어별 색상(정확/부정확)** + 틀린 음소 표시 + 강세 위치 비교(모범 vs 내 발음). 낮은 단어는 탭하면 모범 발음 재생 + 재시도.
- 자체 로직 검토 결과: Web Audio로 피치·음절 길이를 추출해 **강세 패턴 비교(시각화)는 클라이언트에서 가능** — 보조 그래픽으로 채택. 그러나 음소 단위 정확도는 자체 구축이 비현실적(강제정렬+음향모델 필요) → Azure 채택이 결론.
- 모범 음성 TTS 업그레이드 옵션: 기본은 Google Neural2 유지. 발음 진단 모범 음성만 고품질 엔진 검토 — Azure HD($30/1M자)·OpenAI($15/1M자)·ElevenLabs(최고 품질, 고가). 진단 문장은 짧고 캐시되므로 어느 쪽이든 비용 미미. **1차는 Neural2로 충분, 품질 불만 시 교체.**

### 4.9 푸시 알림 · 위젯

- **푸시 알림(웹앱 가능)**: iOS 16.4+부터 홈 화면 PWA도 Web Push 지원. 복습 도착·스트릭 위기·루틴 리마인더. 서버리스 크론으로 발송.
- **위젯(웹앱 불가)**: iOS/Android 위젯은 네이티브 전용 → **Capacitor 전환 시** WidgetKit으로 구현. 콘셉트는 확정해둠: **연속 학습일에 따라 위젯 배경(캐릭터 상태)이 변화** — 성장 스테이지(4.10)와 동일 에셋 재사용.

### 4.10 동기부여 시스템 (저공수 우선)

채택 (디자인·개발 공수 최소 순):

1. **캐릭터 성장 스테이지** — 누적 학습·스트릭에 따라 홈 캐릭터 이미지가 5단계로 변화. 유령의 "NO ENGLISH" 팻말 문구가 점점 바뀌는 서사(예: NO ENGLISH → OH... ENGLISH? → OK ENGLISH). 개발 = 이미지 스왑 로직뿐, 디자인 = 5컷. **공수 대비 효과 최고.**
2. **뱃지 + 스카우트 조끼판** — 조건 달성 시 뱃지 획득, 보관함의 "조끼" 화면에 부착(보이스카우트 방식). 개발 = 조건 체크 + 그리드 화면. 뱃지 아이콘은 사용자(디자이너) 강점 활용. 예: 첫 회화 완주, 7일 연속, 산출 어휘 1천 개, 어원 30개 탐험, SOS 없이 대화 완주.
3. **CEFR 승급 세리머니** — 레벨 테스트 연동, 승급 시 전용 화면 1장(시골쥐→서울쥐 연출). 개발 = 조건 + 화면 1개.

보류(공수 큼): 집 꾸미기/이사, 아이템 착용 꾸미기 — 에셋 조합 폭발로 디자인 공수가 계속 늘어나는 구조라 제외. 선행 어휘앱(국내 대표)식 "꾸미기"의 동기부여는 1~2번 조합으로 대체.

### 4.11 흘려듣기 확장 — 내 대화 복습 재생

기능: 지금까지의 회화·작문에서 **AI가 수정해준 문장들을 한국어→영어 순으로 연속 재생** (기존 흘려듣기의 재생 대상에 "내 대화" 소스 추가).

**학습 효과 검증 결론: 조건부로 유효.**

- 근거 ①: 자기가 말하려다 실패한 문장은 일반 예문보다 기억 우위(자기참조 효과 + 산출 시도 후 정답 노출 = 인출 실패 후 피드백 학습).
- 근거 ②: "내 발화 → 교정 확인" 사이클의 반복 노출은 출력 가설(Swain)·주목 가설(Schmidt)이 말하는 "갭 인지"를 강화 — 교정을 받은 직후 1회로 끝나는 게 보통인데, 간격을 두고 재노출하면 SRS 원리가 문장 단위에 적용됨.
- **유효 조건 3개** (설계 반영):
  1. **수정본(정문)만 재생** — 내 오류 문장을 다시 들려주지 않는다(오류 재각인 방지).
  2. **기본 순서는 한→영** — 한국어를 먼저 듣고 영어를 속으로 인출한 뒤 정답을 듣는 구조(인출 연습). 영→한 순서는 섀도잉·복습용 옵션.
  3. 문장 사이 **2~3초 인출 간격**(무음) 삽입.
- 한계도 명시: 순수 수동 듣기 단독의 효과는 근거가 약함 → 이 기능은 "이동 시간에 하는 보조 복습"으로 포지셔닝하고, 잔디 반영은 안 함(수동 소비를 학습 실적으로 계산하지 않음).

### 4.12 타깃 특화 기능 — "수능은 했는데 회화가 안 되는 사람"

타깃 정의: **기초 문법·독해·듣기는 되는데(수능 학습 경험) 말하기 산출 경험이 0인 학습자.** 문제는 지식 부족이 아니라 ① 수동 지식이 입으로 안 나오고 ② 직역·문어체(수능체)로 말하며 ③ 자연 속도 연음을 못 알아듣고 ④ 막히면 침묵하는 것. 이에 맞춘 특화 기능:

1. **아는 단어 → 쓰는 단어 전환 훈련**: 레벨 테스트를 "인지(뜻 알기)"와 "산출(한국어 보고 말하기)"로 이원 측정 → **"알지만 못 쓰는 단어" 큐**를 별도로 만들어 산출형(한→영 스피킹) 카드로 우선 복습. 통계에 수동/능동 어휘량을 따로 표시.
2. **문법 자동화 드릴**: 문법을 새로 가르치지 않고(이미 앎) **아는 문법을 입으로 자동화** — 패턴별 순간영작 스피킹(가정법, 관계사, 분사구문 등 수능 문법 용어로 분류해 친숙하게). 문법 태그·설명도 수능 용어 체계로 표기.
3. **수능체 감지기**: 작문·회화 첨삭에서 문법 오류와 별도로 **"문어체/직역 티"를 하이라이트** — 격식 지수 표시 + 원어민 구어 버전 제시 (예: "However, I think that..." → "But honestly, ..."). 콩글리시 직역 패턴 사전 내장.
4. **연음·축약 듣기 훈련**: 딕테이션에 또박또박/자연 속도 토글, gonna·wanna·연결발음 구간 하이라이트. 수능 듣기 속도와 실제 회화 속도의 간극을 명시적으로 훈련.
5. **회화 SOS 버튼**: 대화 중 막히면 침묵 대신 SOS → 하고 싶은 말을 한국어로 말하면 영어 표현을 알려주고 **다시 말해보게 한 뒤** 대화 재개. "막히는 경험"을 학습 이벤트로 전환.
6. **필러·리액션 팩**: Well / Let me think / That makes sense 같은 대화 윤활유 표현을 초기 학습 큐에 배치하고, 회화 피드백에서 침묵 구간에 필러 사용을 코칭.
7. **회화 난이도 커브**: 목표가 정해진 롤플레이(입국심사 통과하기 등 스크립트형) → 반자유 대화 → 자유 대화 순. 응답 시간도 무제한 → 제한 모드로 점진 강화.

### 4.13 유창성 확장 — 언어학 검토 채택분 (근거: docs/LINGUISTICS.md)

언어학 검토 결론: 기존 설계는 산출 중심이라 뼈대는 적합하나 **"정확하게"에 편중, "빠르게(자동화)·회복력 있게(협상)"가 부족.** Nation의 Four Strands 중 유창성 개발 가닥을 아래 7건으로 보강한다.

1. **청크(정형표현) SRS — 최우선**: "be alert to", "I was wondering if" 같은 콜로케이션·문두 프레임을 단어와 동급의 카드 단위로. 유창성은 단어 조립이 아니라 통짜 표현 인출에서 나온다(Pawley & Syder). 학습 카드 빈칸이 이미 구 단위라 데이터 확장으로 구현.
2. **섀도잉 모드(듣기 세션 내)**: 문장 듣고 즉시 겹쳐 말하기 5~10문장 연속 세트 — 선언지→절차지 자동화(DeKeyser). 발음 진단 파이프라인 재사용.
3. **다시 말하기 드릴(4/3/2 경량판)**: 회화 종료 후 내 파트를 수정본 기준 60초→40초 재발화. 대화 로그 재활용이라 콘텐츠 비용 0.
4. **어순 스피드 드릴 승격**: 청크 배열 + 반응시간 기록. SOV→SVO 자동화를 계량(레벨테스트 Part B-3과 데이터 공유).
5. **의미협상 페르소나**: 페르소나가 확률적으로 되묻기("Sorry, what do you mean?") — 실전 회복력 훈련(Long). 불친절 페르소나는 되묻기 빈도 상향.
6. **최소대립쌍 모듈(발음 클리닉)**: r/l·f/p·v/b 듣고 고르기 + 말해서 통과. Azure 음소 점수로 채점.
7. **회화 피드백 2모드**: 4.6에 반영(정확성/유창성 모드).

### 5.1 네이밍 후보 (역발상·팝 콘셉트)

| 영포자 | 국내 타깃에 직관 최강, 검색 유리 | 한국어라 확장성 없음 — 서브타이틀로 활용 권장 |
| Pardon? | 회화 정체성, 물음표가 로고 포인트 | 일반 단어라 상표 약하고 스토어 검색 불리 |

- **1안: "NoEnglish — 영포자 탈출"** (영문 브랜드 + 국내용 서브타이틀 병기). Pardon? / Umm.. 은 페르소나·화면 문구(빈 상태 등)에 재활용.
- 확정 전 체크: 앱스토어 동명 앱, 상표검색(KIPRIS), 도메인/SNS 핸들.

### 5.2 캐릭터 시스템 (메인 + 서브, 선행 게임화 학습앱 구조)

- **메인 후보: 유령** — "NO ENGLISH" 팻말을 든 유령. 네이밍 1안과 세계관 일치. 학습이 쌓일수록 팻말 문구·유령의 표정/형체가 변하는 성장 서사(5.4 동기부여와 연결).
- **서브 캐릭터**: 두더지, 상자 속 길고양이, 나무늘보, 시골쥐→서울쥐 — 페르소나 아바타·뱃지·빈 상태 일러스트·카테고리 마스코트로 배치. 시골쥐→서울쥐는 그 자체가 성장 서사라 레벨 승급 연출에 적합.
- **그래픽 스타일**: 원색 + 라인(아웃라인) 벡터 일러스트. 제작: 사용자.
- 사용자 추가 페르소나(4.6)의 프로필 이미지는 이 기본 캐릭터 세트에서 선택.

#### 5.2.4 캐릭터 선택 & 변경 — 포인트 잠금

- **처음 1회는 무료 선택.** 홈에 캐릭터가 없으면 "캐릭터를 선택해 주세요" 카드가 뜨고, 설정에서 고른다.
- **이후 변경은 포인트 소모형**(300P, `CHAR_CHANGE_COST`) — 자유 선택 UI를 홈에서 제거하고 **설정(기어 아이콘) 화면으로 이동**, "일정 학습량 유지"를 계량하기 위해 단어카드 정답 시에만 10P 적립(`POINTS` 엔진, localStorage). 포인트 부족 시 "0/300P"로 비활성 표시.
- 근거: 홈은 매일 보는 화면이라 즉시 바꿀 수 있는 피커가 있으면 습관적 이탈 지점이 되지만, 설정 안에 포인트 게이트로 넣으면 "캐릭터 교체"가 꾸준한 학습에 대한 보상 이벤트로 전환된다.

#### 5.2.5 캐릭터 아트 에셋 파이프라인

- 쥐 캐릭터 러닝 모션은 `img/rat_run.js`(4프레임 SVG, svgo로 52.7% 최적화)로 자산화하고 `SpriteChar` 컴포넌트가 프레임을 순환 재생(연속일 단계에 비례한 fps). 나머지 캐릭터(유령·두더지·외계인)는 실제 아트 나오기 전까지 텍스트 placeholder 유지.
- 새 캐릭터 아트 추가 시: 4프레임 SVG를 같은 방식(`window.CHAR_<NAME>_RUN` 전역 + svgo 최적화)으로 `img/`에 넣고 `SpriteChar` 분기만 추가하면 됨 — 아트 교체가 코드 구조 변경 없이 가능.

#### 5.2.1 캐릭터 로스터 & "달리는 홈" 모션

홈 첫 화면에서 **캐릭터가 계속 달려가는 모션**(제자리 달리기 + 배경 스크롤 = 전진 착각)이 핵심 정서. 선택 캐릭터 4종, 각자 달리는 방식이 다름:

| 캐릭터 | 달리기 표현 |
|---|---|
| 고양이 유령 | 스케이트보드 |
| 두더지 | 땅 파며 질주 |
| 쥐 | 짐보따리 메고 달리기 |
| 외계인 | UFO 타고 |

- **연속 학습일 → 성장 4단계**: 1단계(<3일) · 2단계(<7일) · 3단계(<14일) · 4단계(14일+). 단계가 오를수록 **달리는 속도(모션 빠르기)도 상승**.
- 구현 슬롯: `index.html`에 달리는 씬 뼈대 완성(도로 스크롤·속도선·까딱임 placeholder, 속도는 단계별 CSS 변수). **실제 캐릭터 아트만 끼우면 됨.**

#### 5.2.2 애니메이션 기술 선택 (로띠 필수 아님)

- "다리가 실제로 움직이는 달리기"는 **단일 이미지 CSS로는 불가** → 프레임 기반 필요.
- **1순위 권장: Rive** — 상태머신 하나로 "성장 4단계 + 달리기 속도 + 캐릭터별"을 파라미터(연속일)로 전환. 파일·로직 관리 최적.
- **2순위: Lottie** — 단계별 4개 JSON(After Effects+Bodymovin). 라이브러리 lottie-web(~250KB).
- **무라이브러리 대안**: 스프라이트 시트(steps() 애니) 또는 APNG/애니 WebP. 의존성 0.
- 결론: **로띠는 선택지 중 하나일 뿐 필수 아님.** 4캐릭터×4단계×달리기의 상태 관리를 생각하면 Rive가 가장 깔끔. 어느 쪽이든 현재 만든 씬 슬롯에 그대로 삽입.

#### 5.2.3 위젯 — 교통표지판 (연속일 상태 머신)

홈 스크린 위젯(네이티브, Capacitor 전환 시 WidgetKit). 연속 학습일을 **교통표지판 은유**로:

| 상태 | 표지 | 조건 |
|---|---|---|
| 위험 | **경고 삼각형(!)** | 연속 0일 / 오늘 미학습으로 스트릭 끊길 위기 |
| 정상 주행 | **속도제한 원형(숫자=연속일)** | 학습 유지 중 — 숫자가 커질수록 "빨리 달리는 중" |
| 상위 단계 | 속도제한 숫자 상승 + 캐릭터 달리기 빨라짐 | 성장 단계와 연동 |

- 앱 내 홈 스테이지 좌상단에 **동일 표지를 프리뷰**로 표시(위젯 미출시 단계에서도 콘셉트 노출, 에셋 재사용).
- "속도"와 "달리기"가 한 은유로 묶임 — 캐릭터 속도·위젯 속도제한·성장 단계가 모두 연속일 하나로 구동.

### 5.3 UX 구조 — 하단 탭 5개, 홈 중앙 배치

**학습 루틴 정의: 단어카드 / 듣기 / 회화(챗봇+작문)** — 홈은 이 3단계 루틴의 허브.

| 탭 (좌→우) | 내용 |
|---|---|
| **학습** | 단어카드 세션 · **듣기 세션(딕테이션)** · 퀴즈 · 흘려듣기 · 레벨 테스트 |
| **사전** | **진입 시 허브(메뉴)** — 목록을 쏟아내지 않음: ① 새 단어 등록 ② 내가 학습한 단어 ③ 카테고리별 보기 ④ 즐겨찾기 ⑤ 내 단어장. 검색은 상시 최상단(입력 즉시 결과 전환). **카테고리 솔팅 결론: 1차 레벨(CEFR — 학습 우선순위와 직결), 2차 어근별(타고타고 강점), 3차 주제별(비즈니스/여행 — 시드에 topic 필드 추가 후)** |
| **홈 (중앙)** | 캐릭터 + 스트릭 + **내 레벨 상시 표시**(CEFR·추정 어휘량·상위%) + 오늘의 루틴(단어 → 듣기 → 회화) |
| **회화** | **메신저형 대화방 목록** — 페르소나 방 + 고정 방(작문 연습·발음 클리닉) + 페르소나 추가 |
| **보관함** | 즐겨찾기 단어장(단어/문장) · 뱃지 조끼 · 성장 기록 |

- 통계&설정은 홈 우상단 프로필 진입 유지.
- **5번째 탭 검토 결과 — "사전" 채택.** 근거: ① "없는 단어 추가를 어디서 하나"와 "학습한/전체 단어 목록 보기"가 한 화면의 같은 행위(단어 탐색)라 탭 하나로 해결 ② 사전(전체 DB·탐색)과 보관함(내가 고른 것)의 역할이 겹치지 않음 ③ 타고타고(어원·어근 네트워크)의 자연스러운 입구. 탈락 후보: 통계(프로필 진입으로 충분, 매일 볼 화면 아님), 발음(회화 탭 내 도구가 자연스러움), 랭킹(1인 앱에 무의미).

**단어 상세 네비게이션 (JP 앱과 동일)**: 목록/카드에서 단어 탭 → **팝업(바텀시트)** → 팝업 안에서 유의어·어원·예문 단어 등 타고 들어가면 **모달 스택**으로 쌓이고 뒤로가기로 한 장씩 복귀. 어디서든 동일 패턴(학습 카드·사전·보관함·회화 말풍선).

### 5.3.1 회화 탭 — 메신저 UI

- 탭을 열면 **대화방 목록**(카카오톡형): 페르소나별 방 — 아바타 + 이름 + 마지막 메시지 미리보기 + 이어하기 표시.
- 상단 고정 방 2개: **작문 연습**(입장 시 카테고리 선택 → 문제·첨삭이 메시지로 오가는 방), **발음 클리닉**(수정문 따라 말하기·진단이 메시지 흐름으로 진행).
- 방 입장 = 기존 대화 이어짐(페르소나가 이전 대화 기억 — 관계 서사 유지). 새 대화 시작은 방 메뉴에서.
- 페르소나 추가(+)는 목록 우상단 — 월 1명 제한(4.6).

### 5.4 디자인 방향 (디자인은 사용자가 직접 작업)

- **라이트 모드 온리**, iOS·토스 계열의 정제된 룩: 흰 배경 + 큰 볼드 타이포(숫자 강조) + 라운드 카드 + 넉넉한 여백 + 그림자 최소 + 포인트 컬러 1개(토스 파랑 그대로는 피하고 자체 브랜드 컬러).
- 마이크로 인터랙션(카드 전환·정답 피드백)은 토스식 스프링 모션 소량만.
- **캐릭터(마스코트)**: 선행 게임화 학습앱처럼 사용자가 직접 개발. 앱은 캐릭터 슬롯(온보딩·완료 화면·빈 상태·페르소나 아바타)만 설계해두고 이미지 교체 가능하게. JP 앱의 tsukun 이미지 슬롯 구조 재사용.
- 선행 어휘앱(국내 대표) 차별화 기준은 `COMMERCIAL.md` 부록 B 적용 (알약 빈칸·아이콘 5행·트로피 진행바 재현 금지).

## 6. 개발 타임라인 — 1개월 (개발: Claude Code, 디자인·캐릭터: 사용자)

전제: 개인용 웹앱(PWA) 완성이 목표. 상용화 백엔드(COMMERCIAL.md 부록 A, +4~7주)는 별도.

| 주차 | 개발 (Claude) | 사용자 할 일 (병행) |
|---|---|---|
| **1주차 — 코어** | 시드 파이프라인(CEFR-J Wordlist + WordNet 자동 추출 + 한국어 뜻·예문 직접 작성) · JP 앱 포크 → 학습 카드(빈칸 타이핑/음성 입력) · SRS · TTS 프록시 · 잔디 · 통계 골격 · 임시 디자인 토큰(토스풍) | 무드보드·컬러·타이포 확정 |
| **2주차 — 단어 네트워크 + 퀴즈** | 단어 상세(유의어/반의어/어원/어근/품사 파생, 타고타고) · 어근 탐색 · WordNet·etymology-db 가공 · 수동 추가 · 외부사전 링크 · 단어/문장 퀴즈(빈칸·딕테이션, 연음 토글) · 흘려듣기 | 카드·홈 화면 시안 → 주말에 1차 반영 |
| **3주차 — 회화력** | 작문 첨삭(카테고리·자연스러움%·수능체 감지·즐겨찾기) · 회화 엔진(페르소나 10종, 피드백 게이트, SOS 버튼, 음성 파이프라인) · 단어장(문장·문법 태그) | 캐릭터·페르소나 아바타 10종 제작 |
| **4주차 — 레벨테스트 + 마감** | 레벨 테스트(인지/산출 이원 측정, 어휘량·상위%) · 온보딩 · 필러 팩·산출 큐 · 디자인 시안 최종 반영 · 캐릭터 통합 · PWA/캐시/성능 · 버그픽스 · 배포(Vercel) | 최종 시안 전달 · 전체 검수·실기기 테스트 |

리스크 버퍼: 3주차 회화 음성 파이프라인(iOS 제약 폴백)이 최대 변수 — 밀리면 페르소나를 3종으로 줄여 출시하고 나머지는 데이터 추가로 해결(페르소나는 데이터 정의라 코드 변경 불필요).

v2 추가 기능 배치: **발음 진단(Azure)·사용자 페르소나 → 3주차**, **동기부여(캐릭터 성장·뱃지)·푸시 알림·흘려듣기 대화 소스 → 4주차**. 위젯은 네이티브(Capacitor) 전환 시점의 별도 작업.

## 6.5 개발 전략 — 처음부터 네이티브 vs PWA 후 전환

전제: 개발은 Claude(원격), 코드 자산은 JP 앱(웹), 네이티브로 가더라도 현실적 선택지는 Capacitor(웹 코드 래핑)이므로 **두 경로의 코드는 사실상 동일하다. 진짜 질문은 "언제 래핑하느냐"뿐.**

| 비교 항목 | 처음부터 네이티브 (Capacitor day 1) | PWA 먼저 → 전환 |
|---|---|---|
| 개발 속도 | 매 수정마다 Xcode 빌드→기기 설치/TestFlight 업로드 대기. **Mac 필수**(원격 개발 환경에선 CI 우회 필요) | **수정 → 푸시 → 폰에서 URL 새로고침.** 마찰 0 |
| 초기 비용 | 첫날부터 연 $99 | $0. 계속 쓸 앱인지 검증된 뒤에 지출 |
| 배포·수정 반영 | TestFlight 빌드 처리·심사 대기 | 즉시 |
| STT(회화 말하기) | 네이티브 음성인식 무료 사용 | 폴백(녹음+Gemini 전사, 분당 $0.002)로 해결 — 어차피 회화는 LLM 호출 필요라 자연스러움 |
| 푸시 알림 | 안정적(APNs) | 가능(iOS 16.4+ 웹푸시), 신뢰성은 APNs보다 낮음 |
| 위젯 | 가능 | **불가** — 유일한 실질 결손 |
| 백그라운드 오디오(흘려듣기) | 안정적 | 대체로 동작, 간헐적 끊김 가능 |
| 데이터 안정성 | 앱 저장소 | localStorage 퇴거 리스크 → 백업/복원 기능으로 방어(JP 앱에 이미 있음) |
| 전환 비용 | — | Capacitor 래핑 + 푸시 APNs 교체 + 위젯 + IAP ≈ **1~2주** |

**판정: PWA 먼저.** 근거:

1. 이 프로젝트의 최대 리스크는 기술이 아니라 **"내가 계속 쓰는 앱인가"(리텐션)**. 검증을 가장 빨리 시작하는 경로가 PWA다.
2. 원격 개발(Claude) + 아이폰 확인 루프에서 PWA만 마찰이 없다. 네이티브 루프는 Mac/CI/TestFlight 대기가 매 수정마다 붙어 1개월 타임라인이 깨진다.
3. 처음부터 네이티브로 가서 아끼는 것은 전환 작업 1~2주뿐인데, 그 대가로 개발 기간 내내 느린 루프 + 첫날 $99 + Mac 의존을 진다.
4. PWA 제약 중 진짜 불가는 위젯 하나이고, 위젯은 v1 핵심 기능이 아니다(성장 스테이지 에셋 재사용으로 전환 시 저비용 추가).

**처음부터 네이티브가 정당화되는 조건**(해당 시 재검토): 위젯이 핵심 훅일 때 · 스토어 판매가 이미 확정일 때 · 흘려듣기(백그라운드 오디오)가 주 사용 시나리오의 절반 이상일 때.

전환 트리거: 1개월 완성 후 **2~4주 실사용에서 주 4일 이상 쓰게 되면** Capacitor 래핑 + TestFlight 착수(COMMERCIAL.md 권고 경로와 동일).

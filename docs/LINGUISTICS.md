# 언어학적 검토 — "책으로 배운 사람이 실제 회화를 하게 되는 앱인가?"

작성일: 2026-07-11 · 대상: PLAN.md v2 + wireframe v3 전체
검토 질문: **문법·독해·듣기는 되지만 말하기 산출 경험이 0인 학습자(수능 세대)가 이 앱으로 실제 회화 가능 수준에 도달할 수 있는가?**

---

## 1. 전제 — 한국어 화자는 왜 유럽인보다 오래 걸리는가

이 앱의 설계가 맞는지 판단하려면 먼저 "무엇이 병목인지"를 정확히 놓아야 한다. 한국인의 fluency 지연은 게으름이나 교육열 부족이 아니라 **구조적 요인 5가지의 중첩**이다.

### 1.1 유형학적 거리 — 출발선 자체가 다르다

- 미 국무부 FSI 분류에서 영어 화자 기준 한국어는 **최고 난도 Category IV(약 2,200시간)**, 프랑스어·네덜란드어는 Category I(약 600~750시간). 이 거리는 대칭적이다 — 한국어 화자가 영어에 도달하는 데도 유럽인의 **3~4배 시간**이 구조적으로 필요하다.
- 구체적 격차: ① **어순** SOV↔SVO (모든 문장에서 실시간 재배열 필요) ② **형태론** 교착어(조사)↔굴절·고립어(관사·전치사 — 한국어에 대응물 없음) ③ 관사·수일치·시제상(perfect/progressive) 체계 부재 ④ 관계절이 명사 앞(한국어) vs 뒤(영어).
- **동족어(cognates) 부재**: 네덜란드인은 영어 어휘의 상당 부분을 "이미 아는 단어의 변형"으로 만난다. 한국인에게 영어 어휘는 거의 전부 신규 암기다. 어휘 습득 부담이 근본적으로 다르다.

### 1.2 음운·리듬 격차

- 음소 대립 부재: /r/-/l/, /f/-/p/, /v/-/b/, /θ/-/ð/, /z/-/dʒ/ — 듣기 단계에서부터 구분이 안 되면 산출은 불가능하다.
- **리듬 체계 충돌**: 한국어는 음절박자(syllable-timed), 영어는 강세박자(stress-timed). 영어의 축약·연음(gonna, didja)은 강세박자의 필연적 결과인데, 수능 듣기는 또박또박 읽어주므로 이 격차가 시험으로는 드러나지 않다가 실전에서 터진다.

### 1.3 교육 방식 — 명시지만 쌓이고 절차지가 없다

- 한국 공교육은 문법번역식(GTM) 중심: 규칙을 **설명할 수 있는 지식(declarative)**은 세계 최상위권으로 쌓이지만, 실시간으로 **꺼내 쓰는 자동화(procedural)**로 전환하는 훈련이 없다. 스킬 습득 이론(DeKeyser)에서 이 전환은 **산출 반복**으로만 일어난다.
- 입력 편중: 수능·토익은 100% 수용 기능(읽기·듣기)만 측정 → 학습 동기·시간이 전부 입력에 투자된다. Swain의 **출력 가설(Output Hypothesis)**: 이해 가능한 입력만으로는 유창성이 생기지 않고, 산출을 시도해야 내 문법의 구멍(gap)을 인지하고 메운다.

### 1.4 정의적(affective) 요인

- 오류 회피·체면 문화 + "완벽한 문장이 아니면 말하지 않는다" → Krashen의 **정의적 여과(affective filter)**가 높아 입력·산출 모두 차단. 침묵이 길수록 자동화는 더 늦어지는 악순환.
- 유럽 학습자는 계통적 유사성 덕에 초기 성공 경험이 빠르고, 그 성공이 발화 의지(WTC, willingness to communicate)를 강화한다. 한국 학습자는 초기 실패 경험이 발화 회피를 강화한다.

### 1.5 노출 환경

- 유럽: 더빙 없는 TV·관광·인접 언어 사용자와의 일상 접촉 = 준(準) 제2언어 환경.
- 한국: 순수 EFL 환경 — 교실 밖 영어 사용 필요가 0에 수렴. **주당 유의미한 산출 기회가 0시간**인 학습자가 대부분.

### 종합

한국 학습자의 병목은 "지식 부족"이 아니라 **① 산출 자동화 부재 ② 어순 재배열 부하 ③ 리듬·연음 청취 격차 ④ 높은 정의적 여과 ⑤ 산출 기회 0**이다. 앱이 이 5개를 정조준하는지가 평가 기준이 된다.

## 2. 이 앱의 적합성 평가 — 무엇이 맞고, 무엇이 비어 있나

### 2.1 이론과 정합적인 설계 (유지·강화할 것)

| 앱 설계 | 병목 대응 | 이론적 근거 |
|---|---|---|
| 루틴 3단계가 전부 산출 포함(빈칸 타이핑 → 딕테이션 → 회화) | ①⑤ | Output Hypothesis — 입력 소비형 앱과의 결정적 차별점 |
| 인지/산출 어휘 이원 측정 + "알지만 못 쓰는 단어" 큐(4.12-1) | ① | Nation의 수용/산출 어휘 구분. 수능 세대의 핵심 격차를 직접 계량 |
| 회화 SOS 버튼 + AI 상대(체면 손상 없음) + 관계 페르소나 | ④ | 정의적 여과 하강, WTC 상승 — AI 대화 상대의 검증된 강점 |
| 피드백 게이트("OK 눌러야 진행") | ① | 노티싱 가설(Schmidt) — 오류를 인지해야 수정된다 |
| 연음/자연 속도 토글(딕테이션) | ③ | 강세박자 리듬 훈련. 수능 듣기와 실전의 간극을 명시적으로 다룸 |
| 수능체 감지기 | ①(질적) | 문어/구어 레지스터 인식 — 이 타깃에 특화된 독창적 장치 |
| SRS + 문맥(예문) 어휘 + 어원 네트워크 | 어휘 부담(1.1) | 간격 반복 + 정교화(elaboration). 동족어 부재를 어원 연결로 부분 보상 |
| 흘려듣기 "내 대화 수정본, 한→영 순서" | ① | 인출 연습 구조 — 이미 4.11에서 조건부 설계 완료 |

**판정: 뼈대는 적합하다.** 특히 "산출 중심 루틴 + 산출 어휘 계량 + 저부담 대화 상대" 조합은 시중 앱(입력 소비형 다수)보다 이 타깃에 잘 맞는다.

### 2.2 비어 있는 것 — 언어학적 격차 7건

Nation의 균형 잡힌 커리큘럼(Four Strands: 의미 중심 입력 / 의미 중심 출력 / 언어 형식 학습 / **유창성 개발**) 기준으로 점검하면, 앞의 셋은 충실하나 **넷째 가닥(유창성 개발 = 아는 것을 빠르게 쓰는 훈련)이 거의 없다.** 이 결핍이 "책으로 배운 사람" 문제의 본질과 정확히 겹치므로 반드시 보완해야 한다.

**G1. 청크(정형표현) 단위 학습 부재 — 최우선.**
원어민다운 유창성은 단어 조립이 아니라 **통짜 표현(formulaic sequences)의 인출**에서 나온다(Pawley & Syder). 단어 SRS만 있으면 학습자는 매 문장을 단어에서 조립하며 어순 부하를 정면으로 맞는다.
→ **채택: 청크를 SRS의 1급 단위로.** "be alert to", "It depends on", "I was wondering if" 같은 콜로케이션·문두 프레임을 단어와 동급 카드로 학습. 학습 카드의 빈칸도 이미 구 단위이므로 데이터 확장으로 구현 가능(코드 변경 최소).

**G2. 섀도잉·음독 루프 부재.**
자동화(1.3)는 구두 반복에서 온다. "말해보기"는 1회 발음 확인이지 자동화 훈련이 아니다.
→ **채택: 듣기 세션에 섀도잉 모드 추가** — 문장을 듣고 즉시 겹쳐 말하기, 5~10문장 연속 세트. 발음 진단(Azure) 파이프라인 재사용.

**G3. 유창성 드릴(4/3/2류) 부재.**
Nation의 4/3/2: 같은 내용을 4분→3분→2분으로 반복 발화하면 정확도를 잃지 않고 속도가 는다. "아는 것을 빨리 꺼내는" 훈련은 이것이 표준이다.
→ **채택(경량화): "다시 말하기 드릴"** — 회화 세션 종료 후, 방금 한 대화의 내 파트를 수정본 기준으로 60초→40초 재발화. 신규 콘텐츠 제작 없이 대화 로그 재활용.

**G4. 어순 자동화가 보조 훈련에 머묾.**
SOV→SVO 재배열(1.1)은 한국인 최대 부하인데 현재 어순 배열은 퀴즈 선택 옵션이다.
→ **채택: 어순 배열을 반응시간 측정 드릴로 승격** — 청크 단위 배열 + 속도 기록. 레벨테스트 Part B(아래)와 데이터 공유.

**G5. 페르소나가 "항상 알아듣는" 문제 — 의미협상 부재.**
실전 회화력은 상대가 못 알아들었을 때의 복구(clarification, 재표현)에서 자란다(Long의 상호작용 가설). 항상 이해해주는 AI는 이 근육을 못 만든다.
→ **채택: 페르소나가 확률적으로 "Sorry, what do you mean?"을 던지도록** 프롬프트 설계. 특히 불친절 페르소나(Karen)는 되묻기 빈도를 높게 — 기존 페르소나 콘셉트와 자연스럽게 결합된다.

**G6. 음소 대립(minimal pairs) 훈련 부재.**
r/l, f/p, v/b를 못 듣는 상태에서 발음 점수만 주면 학습자는 뭘 고쳐야 할지 모른다.
→ **채택: 발음 클리닉에 최소대립쌍 모듈** — 듣고 고르기(rice/lice) + 말해서 통과하기. Azure 음소 점수로 채점 가능.

**G7. 피드백 게이트의 부작용 — 유창성 흐름 절단.**
매 발화 교정+OK는 노티싱에는 좋지만, **대화 흐름을 계속 끊으면 "정확성 모니터 과잉"(Krashen의 monitor over-use) — 정확히 수능 세대가 이미 가진 병을 강화할 위험**이 있다.
→ **채택: 회화 피드백 2모드** — ① 정확성 모드(현행: 즉시 게이트) ② **유창성 모드(대화 중 무중단, 종료 후 일괄 리뷰)**. 초기엔 정확성 모드로 시작하되 주 1회 이상 유창성 모드를 루틴에 섞도록 권장. 이것이 이번 검토의 가장 중요한 수정이다.

### 2.3 채택 요약 (기획 반영)

G1 청크 SRS · G2 섀도잉 모드 · G3 다시 말하기 드릴 · G4 어순 스피드 드릴 · G5 의미협상 페르소나 · G6 최소대립쌍 · G7 피드백 2모드 → **PLAN.md 4.13에 신설 반영.** 개발 비용은 대부분 데이터·프롬프트 수준이며(G2·G6만 화면 추가), 기존 파이프라인(Azure·TTS·대화 로그)을 재사용한다.

## 3. 결론

**"적합한가?" — 뼈대는 그렇다. 산출 중심 루틴과 산출 어휘 계량은 이 타깃의 병목을 정확히 겨눈다.**
다만 검토 전 설계는 "정확하게 말하기"에 편중되어 있었다. 책으로 배운 학습자에게 부족한 것은 정확성이 아니라 **속도(자동화)와 회복력(협상)**이므로, 2.2의 7개 — 특히 청크 단위 학습(G1)과 유창성 모드(G7) — 를 넣어야 "아는 영어"가 "나오는 영어"로 전환되는 완결 루프가 만들어진다.

기대 관리 한 줄: 이 앱이 채우는 것은 **산출 기회 0 → 매일 20~30분의 저부담 산출**이다. FSI 시간(1.1)이 말하듯 절대 시간은 지불해야 하며, 앱의 역할은 그 시간을 "한국인의 병목 5개에 정조준된 시간"으로 만드는 것이다.

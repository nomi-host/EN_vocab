# 베타테스트 오픈 전 체크리스트 (2026-07-23 갱신, v0.35.100 기준)

v0.35.91에서 시작한 베타 전 점검이 v0.35.100까지 10차례 반복됐다(v0.35.99는 이 문서 작성 중
다른 세션이 동시에 배포한 별개 수정 — 잘린 Gemini JSON 응답 복구/재시도, 아래 목록엔 포함 안 됨).
아래는 전체 누적 요약 —
✅ = 코드 레벨 완료+Playwright 실렌더 검증 완료 / ☐ = 오픈 전 사람이 직접 확인 필요.

## 1. 이번 세션에서 잡은 버그·기능 전체 (v0.35.91 → v0.35.98, v0.35.100)

**치명(앱이 아예 안 되던 것)**
- ✅ 회화 "전송" 버튼 탭 100% 크래시(`onClick={send}` 이벤트 객체 인자 버그) — 모바일 회화 학습 자체가 불가능했던 원인.
- ✅ React/Babel unpkg CDN 직결로 인한 백지 화면 — `lib/`에 자체 호스팅으로 전환.

**높음**
- ✅ 계정 삭제 API CORS 차단(Authorization 헤더 누락).
- ✅ 로그아웃/계정삭제 후 다른 계정에 학습기록 혼입 가능(en_srs_bak 미삭제).
- ✅ 회화 전송 후 "먹통"(매 턴 확인 게이트로 입력창 잠김) — 자동 진행으로 변경.
- ✅ 회화 입력창 레이아웃 — 두 차례 재보고 끝에 이 앱에서 이미 검증된 `.qbottom`과 동일한
  `max(40px, --kb)` 공식으로 정착. React state 기반이었을 때 전송 버튼 클릭이 자체적으로
  레이아웃을 흔드는 경합까지 있었던 것도 순수 CSS 전환으로 근본 제거.
- ✅ "첨삭을 받아오지 못했어요 (Load failed)" — 네트워크 순간 끊김 자동 1회 재시도로 사용자
  노출 최소화(재현 테스트로 확인: 첫 실패→재시도 성공→에러 미노출).

**중간**
- ✅ 회화·첨삭 응답 지연 — 모델 폴백 웜 캐시 + 짧은 응답 호출의 토큰 예산 축소(2048→1024).
- ✅ 유저 말풍선 우측 꼬리 잘림, 딕테이션 결과 줄간격, 홈 캐릭터 탭 안내 등 UI 다수.
- ✅ Cloudflare `wrangler.toml` 워커명 불일치 정리(`noenglish-app`→실배포명 `no-english`).

**기능 추가(문법 태그 시스템, PLAN 2.5 마지막 미구현분)**
- ✅ 회화·작문·딕테이션·보관함·레벨테스트(Part B 결과) 5개 화면 전체에 문법 칩 확장.
- ✅ 보관함 "같은 문법으로 저장된 다른 문장" 필터(4.7절 오래 미구현 항목).
- ✅ 문장 속 실제 문법 위치를 형광펜(크림톤)으로 표시 — `detectGrammarSpans` 신설.
- ✅ to부정사 용법 배타 판정 재설계 — 같은 자리에 명사적+부사적이 동시에 붙던 오탐,
  방향구("~에 가서 ~하러") 오분류, 의미상 주어 오탐 3건 모두 실사용 리포트로 발견해 수정.
- ✅ 문법 칩 레이아웃(1열 미적재)·색상(골드→크림)·설명 텍스트 간격 UI 다듬기.

**비용 보호(베타 예산 방어)**
- ✅ **1인당 하루 AI 호출 40회 공유 상한** — 회화·SOS·작문 첨삭·사전 자동조회·발음(STT) 전부
  합산(TTS는 별도 예산이라 제외). localStorage 카운터(`en_gemini_daily_v1`), 자정 지나 새 날짜가
  되면 자동 리셋, 초과 시 "오늘 AI 기능 사용 한도를 다 쓰셨어요" 안내 후 정상 실패 처리(크래시
  없음). 베타 10명·2만원 예산·2~3주 기간 기준 계산(회화/작문/SOS/사전조회 콜당 비용 추정 + STT는
  보수적으로 포함)으로 산정 — 실사용이 매일 상한까지 가진 않을 것이므로 여유 있음. 클라이언트
  단(localStorage)이라 정식 유료 오픈 때는 서버(Supabase) 강제로 교체 필요(현재는 신뢰 기반
  소규모 베타에 한한 임시 방어).
- ✅ **사전 자동조회 공유 캐싱(신규 테이블 `word_cache`)** — 사전에 없는 새 단어를 등록하면
  Gemini로 한국어 뜻·CEFR·예문을 생성하는데, 같은 단어를 다른 베타테스터가 또 등록해도 최초
  1회만 Gemini를 호출하고 이후엔 공유 캐시를 읽는다. `pipeline/supabase_beta_schema.sql` 7번
  섹션 — **오픈 전 이 블록을 Supabase SQL Editor에서 별도 실행해야 함**(아래 4번 사람 확인 항목에
  추가). SB(Supabase 클라이언트)가 없거나 캐시 조회/기록이 실패해도 기존 Free Dictionary→Gemini
  흐름으로 조용히 폴백 — 캐시는 best-effort.
- ✅ **음성 입력 브라우저 내장 인식 우선 사용** — Chrome/Edge/Android처럼 `SpeechRecognition`을
  지원하는 브라우저는 완전 무료·즉시 처리되는 내장 API로 전사하고, iOS/데스크톱 Safari처럼
  미지원 브라우저만 기존 MediaRecorder+Gemini 오디오 전사로 자동 폴백(회화 대화방·발음 클리닉·
  단어카드 음성 입력 3개 지점 모두 적용). 기존 MediaRecorder 코드 경로는 그대로 남겨뒀고
  분기만 추가해 회귀 위험 최소화.

## 2. 실브라우저 자동 검증 통과 항목 (v0.35.100 최종 상태 기준 재확인 완료)

- 전 기능 16개 플로우(단어카드·듣기·사전·회화 SOS/작문/발음클리닉·보관함·설정·레벨테스트)
  **콘솔/페이지 에러 0건**.
- 문법 태그 5개 통합 지점(회화 봇대사·회화 첨삭·작문·보관함·레벨테스트) 전부 칩 렌더+시트
  오픈+"같은 문법 저장 문장" 섹션 정상 확인.
- `detectGrammar`/`detectGrammarSpans` 엣지케이스(빈 문자열·HTML특수문자·이모지·300자+
  장문·중첩 가정법) 전부 예외 없이 처리 확인.
- 네트워크 자동 재시도 실동작 확인(강제 실패→재시도 성공→에러 미노출).
- 회화 입력창 geometry(평소 40px 하한) 확인.
- 일일 AI 호출 상한 40회 정확히 소진 시 차단+안내 문구, STT도 같은 카운터 공유, 날짜 바뀌면
  리셋 확인(Playwright로 fetch mock 40+3회 호출해 정확히 40번째까지만 성공 확인).
- JSX 전체(~9,000줄) Babel 컴파일 + Node 문법 검사 + `api/gemini.js` 문법 검사 통과.
- 이 문서의 모든 항목은 로컬에서 origin/main과 diff 0(내용 완전 일치) 상태로 검증됨.

## 3. 이번 세션이 검증 못 하는 것 (환경 제약 — 솔직한 한계)

- **실제 배포 도메인에 접근 불가** — 이 세션의 네트워크 프록시가 Cloudflare Workers 도메인·
  Vercel 프록시 도메인 등 외부 호스트를 막고 있어(허용목록 방식), `curl`로 라이브 사이트
  버전을 직접 확인할 수 없었음. 로컬 코드가 origin/main과 diff 0인 것까지만 확인됨 — **실제
  배포가 이 커밋을 반영했는지는 아래 3번 항목대로 민영님이 직접 확인 필요**.
- **모바일 Safari/PWA 고유 동작**(visualViewport 키보드 애니메이션, 백그라운드 오디오 세션,
  자동재생 언락, 홈 인디케이터 세이프에어리어) — 헤드리스 Chromium은 이걸 재현 못 함. 이번
  세션의 입력창 여백 수정도 "이 앱에서 이미 실기기 검증된 다른 요소와 같은 공식으로 통일"이
  근거이지, 이 세션 자체가 실기기에서 재검증한 것은 아님.
- **실제 Gemini/Supabase/Google OAuth 왕복** — 전부 목(mock) 응답으로 검증. 실제 API 키·쿼터·
  인증 플로우는 라이브 환경에서만 확인 가능.

## 4. 오픈 전 사람이 직접 확인할 것 (배포/설정 — 이전 회차와 동일, 아직 미확인)

- ☐ **Vercel 환경변수** — `GEMINI_API_KEY`, `GOOGLE_TTS_KEY`, `APP_SHARED_SECRET`,
  `SUPABASE_SERVICE_ROLE_KEY`(계정 삭제용 — 등록 전까지 안전하게 실패). `GEMINI_MODEL`
  변수는 **만들지 말 것**.
- ☐ **Vercel 재배포** — 이번 세션 커밋들(CORS·모델 캐시·maxTokens 등)이 실제 반영되려면
  main 머지 후 재배포 필요.
- ☐ **Supabase** — Auth URL Configuration에 배포 도메인 등록, Google OAuth 동의 화면
  "게시" 상태, RLS 정책 적용 여부, 무료 플랜 활성 상태.
- ☐ **Supabase `word_cache` 테이블 신설(2026-07-23 추가 항목)** —
  `pipeline/supabase_beta_schema.sql` 7번 섹션을 SQL Editor에서 실행. 안 하면 사전 자동조회
  캐싱이 그냥 조용히 스킵되고(에러 없음) 기존 방식(매번 Gemini 호출)으로 동작 — 기능 자체가
  깨지진 않지만 캐싱 이점이 없음.
- ☐ **Cloudflare 배포** — `lib/react*.js`·`lib/babel.min.js`·`lib/compromise.min.js` 등이
  실제로 함께 배포됐는지(빠지면 앱 백지). `앱주소/version.json`이 `v0.35.100` 반환하는지 확인.
- ☐ **Gemini 무료 쿼터** — 베타 인원 동시 사용량 대비 확인.

## 5. 오픈 전 실기기 수동 QA (자동화 불가 — 이전 회차와 동일, 아직 미확인)

- ☐ iOS Safari PWA: 구글 로그인 → 학습 → 강제종료 → 재실행 시 로그인 유지.
- ☐ 기기 2대 동기화(A 학습→백그라운드 업로드→B 로그인 시 복원→B 학습→A "지금 동기화" 시 내려받기).
- ☐ 로그아웃→게스트 상태→재로그인 시 서버 복원.
- ☐ 회화 음성 입력·TTS 재생·백그라운드 음악 안 끊김.
- ☐ **회화 입력창이 키보드 유무에 따라 실기기에서 자연스럽게 붙는지**(이번 세션 세 번째
  수정분 — 코드 공식은 검증된 패턴으로 통일했지만 실기기 최종 확인 안 됨).
- ☐ PWA 업데이트 자동 새로고침 1회(무한 루프 없음).
- ☐ **음성 입력 브라우저 내장/폴백 분기(2026-07-23 추가 항목)** — Android Chrome 실기기에서
  회화·발음클리닉·단어카드 음성 입력이 브라우저 내장 인식으로 빠르게 동작하는지, iOS Safari
  실기기에선 여전히 기존 방식(녹음 후 서버 전사)으로 정상 동작하는지 확인. 이 세션은 헤드리스
  Chromium에서 `SpeechRecognition` API 존재 여부로 분기 로직만 검증했고, 실제 인식 정확도·
  권한 프롬프트 UX는 실기기에서 확인 안 됨.

# 베타테스트 오픈 전 체크리스트 (2026-07-20, v0.35.91 기준)

어제(07-19~20) 보고된 "학습기록 안 보임 · 로그인 풀림 · 회화 느림/에러" 계열 버그의
재발 방지를 위해 코드 전체 리뷰 + 실브라우저 자동 테스트를 돌고 정리한 목록.
✅ = 이번 리뷰(v0.35.91)에서 완료 / ☐ = 오픈 전에 사람이 직접 확인 필요.

## 1. 이번 리뷰에서 잡은 버그 (v0.35.91에 포함)

- ✅ **[치명] 회화 대화방 "전송" 버튼 탭이 100% 크래시** — `onClick={send}`가 클릭 이벤트
  객체를 `overrideText` 인자로 넘겨 `event.trim()` TypeError. Enter 키 전송은 정상이라
  데스크톱에선 안 잡혔고, **모바일은 버튼 탭이 유일한 전송 수단이라 회화 학습이 통째로
  불가능했음**(어제 "회화 에러로 학습 불가" 증상의 직접 원인으로 추정). Playwright 실렌더로
  재현 → 수정 → 재검증 완료.
- ✅ **[치명] React/Babel unpkg CDN 직결** — unpkg 장애·차단 환경(일부 회사망/통신망)에서
  앱이 통째로 백지(root 렌더 0, 실측 재현). lottie/compromise/supabase처럼 `lib/`에 자체
  호스팅으로 전환(react.production.min.js · react-dom.production.min.js · babel.min.js).
- ✅ **[높음] 계정 삭제 API가 브라우저에서 무조건 실패** — `Authorization` 헤더가 CORS
  `Access-Control-Allow-Headers`에 없어 프리플라이트에서 차단(앱은 Cloudflare, API는 Vercel
  크로스오리진). api/gemini.js `setCors`에 추가.
- ✅ **[높음] 로그아웃/계정삭제 후 다른 계정으로 데이터 혼입 가능** — `en_srs_bak`(SRS 로컬
  백업)이 안 지워져서, 다음 부팅 때 SRS 초기화 로직이 백업 복원을 타 이전 사용자의 학습
  기록이 되살아나고 새 계정 서버로 업로드될 수 있었음. signOut·deleteAccount 모두에서 삭제.
  계정 삭제 시 로컬 학습 데이터 전체 정리(게스트 초기화)도 signOut과 동일 범위로 통일.
- ✅ **[중간] 회화·첨삭 응답 지연** — Vercel 프록시가 매 요청마다 폴백 체인의 죽은
  모델(404/429)들을 앞에서부터 다시 두드림. 마지막 성공 모델을 웜 인스턴스에 5분간 기억해
  곧장 살아있는 모델부터 시도하도록 개선.

## 2. 실브라우저 자동 검증 통과 항목 (이번 세션 Playwright)

- 온보딩 → 홈/학습/사전/회화/보관함 5개 탭 전부 콘솔 에러 0으로 렌더.
- 단어카드 세션: 채점 3건 → 홈 "오늘의 루틴 3/20개" · 연속 1일 · `en_activity_v1` 실시간
  반영(= "보관함엔 쌓이는데 홈엔 안 보임" 증상 재발 없음).
- 회화방: 버튼 전송 정상, 네트워크 실패 시 30초 타임아웃 후 에러 문구 + "다시 시도" 노출,
  입력 영구 잠김 없음.
- JSX 전체(≈8,600줄) Babel 컴파일 + Node 문법 검사 통과.

## 3. 오픈 전 사람이 직접 확인할 것 (배포/설정)

- ☐ **Vercel 환경변수** — `GEMINI_API_KEY`(AI Studio 발급 AQ. 키), `GOOGLE_TTS_KEY`,
  `APP_SHARED_SECRET`(index.html `PROXY_SECRET`과 동일 값),
  **`SUPABASE_SERVICE_ROLE_KEY`(계정 삭제용 — 미등록 상태, 등록 전까지 계정 삭제 버튼은
  "server not configured" 안전 실패)**. `GEMINI_MODEL` Variable은 **만들지 말 것**(옛 값이
  코드보다 우선 적용되는 함정).
- ☐ **Vercel 배포 반영 확인** — 배포 후 첨삭 에러 메시지에 "v4" 마커가 보이면 신규 코드.
  이번 CORS 수정(delete-account)과 모델 메모는 Vercel 재배포돼야 적용됨.
- ☐ **Supabase** — ① Auth > URL Configuration에 실제 배포 도메인(리디렉트 허용) 등록
  ② Google OAuth 동의 화면이 "게시" 상태인지(테스트 모드면 등록된 테스터만 로그인 가능)
  ③ `learning_state`/`profiles`/`bug_reports` RLS 정책이 스키마(pipeline/supabase_beta_schema.sql)
  대로 적용돼 있는지 ④ 무료 플랜 일시정지(1주 미사용) 여부.
- ☐ **Cloudflare 배포** — `_headers`가 index.html/version.json/manifest.json에 no-store로
  실제 적용되는지(curl -I로 확인). 새로 추가된 `lib/react*.js`·`lib/babel.min.js`가 함께
  배포됐는지(빠지면 앱 백지 — 이번 변경의 최중요 배포 확인 포인트).
- ☐ **Gemini 무료 쿼터** — 베타 인원 동시 사용량 추정 대비 일일 한도(429 시 폴백 체인이
  버텨주지만 전 모델 소진 가능). 결제 미연결 프로젝트 유지 여부 재확인.

## 4. 오픈 전 실기기 수동 QA (자동화 불가 항목)

- ☐ iOS Safari(홈화면 PWA): 구글 로그인 → 학습 → 앱 강제종료 → 재실행 시 로그인 유지
  (어제 "로그인 풀림"의 재발 확인 — v0.35.86의 토큰 회전 방어가 실기기에서 유효한지).
- ☐ 기기 2대 시나리오: A에서 학습 → 백그라운드 전환(자동 업로드) → B에서 로그인 시 서버
  데이터 복원 → B에서 학습 → A "지금 동기화" 시 내려받기(묵은 쓰기 방지) 동작.
- ☐ 로그아웃 확인 다이얼로그 → 로그아웃 → 게스트 상태(기록 0) → 재로그인 시 서버 복원.
- ☐ 회화: 음성 입력(마이크 권한 → 전사 → 자동 전송), TTS 재생(캐릭터 성별/속도),
  iOS 백그라운드 음악 안 끊김.
- ☐ 듣기 세션·단어카드: 키보드 즉시 팝업, 채점 효과음, 오답 노트 적재.
- ☐ 홈 잔디·연속일이 실제 학습 후 곧바로 갱신되는지(이번 자동 테스트로 데스크톱은 확인,
  실기기 1회 재확인).
- ☐ PWA 업데이트: 구버전 설치 상태에서 새 배포 후 앱 재진입 시 자동 새로고침 1회(무한
  깜빡임 없음).

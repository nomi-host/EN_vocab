### 디자인 토큰 (컬러 · 타이포)

더 이상 플레이스홀더 아님 — `index.html` `:root`에 확정 반영됨.

- **톤앤매너**: 레트로 노트 두들 (옅은 베이지 + 블랙 버튼 + 형광펜 하이라이트)
- **배경/서피스**: `--bg:#F5EEDD`(노트 베이지), `--surface:#FAF4E7`(카드용 크림)
- **텍스트/버튼**: `--ink:#211C15`(잉크 블랙, 텍스트+블랙 버튼 겸용), `--ink-2:#4A4234`, `--muted:#8C8270`
- **포인트 컬러**: 별도 hue 없음 — `--accent`가 `--ink`와 동일(블랙 버튼이 곧 포인트). 토스 블루 회피 기준(COMMERCIAL.md) 충족
- **형광펜 팔레트**: `--hi-green` `--hi-pink` `--hi-yellow` `--hi-blue` 4색, 각각 5단 톤(xlight/light/base/dark/xdark) — 뱃지·태그는 "연한 배경 + 가장진한 텍스트" 조합으로 사용
- **형광펜 하이어라키 규칙**(레벨/난이도 표시 전부 공유): 연두(입문) → 옐로우(중급) → 핑크(고급) → 블루(최상급). CEFR 태그, 카테고리 카드, 뱃지 달성 난이도에 전부 동일 규칙 적용
- **의미 상태**: `--good`(정답/완료), `--danger`(오답) — 형광펜 4색과 분리해 혼동 방지
- **폰트**: LINE Seed KR (Regular/Bold, SIL OFL 1.1 라이선스) — `img/fonts/`에 자체 호스팅, 라이선스 파일 동봉

## 🔜 필요 (우선순위 순)

## 1. PWA 앱 아이콘 — `지금 바로`

- **왜 1순위인가**: `manifest.json`이 이미 `img/icon-192.png`, `img/icon-512.png`를 참조하는데 실제 파일이 없음. 지금 홈 화면에 추가하면 아이콘이 안 뜸.
- **스펙**:
  - `img/icon-192.png` — 192×192px, PNG
  - `img/icon-512.png` — 512×512px, PNG (같은 디자인을 배율만 다르게)
  - (선택, iOS 홈 화면 품질용) `img/apple-touch-icon.png` — 180×180px, 배경색 있어야 함(투명 배경 X)
- **컨셉 제안**: 확정된 배경색(`--bg` #F5EEDD) 위에 블랙 잉크 라인 심볼. 캐릭터가 확정되면 캐릭터 아이콘, 아직이면 앱 심볼(예: 연필/노트 모티프)로 임시 제작 가능.

## 2. 캐릭터 1~4 아트 — `막혀있음, 픽셀 스펙 완성됨`

- **현재 상태**: 기존 "쥐만 실제 그림 + 종(고양이유령/두더지/외계인) 텍스트 placeholder" 구조를 전면 폐기. **종 개념 없이 캐릭터 1~4로 재정의**했고, 1차로 Claude가 원·삼각·사각·꽃 실루엣의 두들 캐릭터를 직접 제작했으나 **사용자가 "메인카드가 못생겨서 개발하기 싫다"고 판단, 임시로 렌더를 꺼둔 상태**(`index.html`의 `CHAR_ART_READY = false`).
  - 코드/로직(`CHARACTERS` 배열, `CharDoodle` 컴포넌트, 뱃지트랙·road·모션라인 애니메이션)은 전부 보존되어 있어 **디자인 파일만 오면 `CHAR_ART_READY = true`로 바꾸고 각 캐릭터 아트만 교체하면 즉시 반영**됨.
  - 지금 화면엔 캐릭터 대신 형광펜 색상 스와치(연두/핑크/옐로우/블루)만 최소 표시 중.
- **정확한 배치 스펙**: **[`docs/stage-pixel-spec.svg`](./stage-pixel-spec.svg)** — 홈 히어로 스테이지의 실제 픽셀 좌표를 라이브 코드에서 직접 측정해 만든 블루프린트. Figma/Illustrator에서 그대로 열어 가이드로 쓸 수 있음. 기준 뷰포트 390px 폭에서:
  - **캐릭터 아트 캔버스**: `140×140px` (정사각형, 이 안에 캐릭터 전체가 들어와야 함)
  - **캐릭터 슬롯(외곽 여유)**: `156×156px` — 달리기 bob 애니메이션(상하 ±7px, 회전 ±1.5°) 여유 포함
  - **뱃지**: `62×62px` 원형, 14px 간격으로 가로 스크롤(카드 우측으로 자연스럽게 잘려 나감)
  - **도로선**: 카드 폭 전체 × 10px, 하단에서 16px 위
  - **모션 라인**: 캐릭터 좌측 뒤, 3개 막대(28/18/24px 폭 × 6px 높이, 8px 간격)
- **스타일**: 벡터, 두꺼운 잉크 아웃라인 + 손그림 러프닝(불규칙한 선 — Illustrator "Roughen" 필터와 유사한 느낌). 몸통 전체를 형광펜 색 중 하나로 채움(캐릭터당 1색: 연두/핑크/옐로우/블루). 입 없음, 눈 2개 + 다리 2개만.
- **파일 형태**: SVG 4개(캐릭터별 1장, 정지 포즈 — 프레임 애니메이션 불필요, CSS가 bob 처리). `docs/stage-pixel-spec.svg`를 배경 가이드로 깔고 그 위에 각 캐릭터를 캔버스 크기(140×140)에 맞춰 그려주시면 코드 수정 없이 치수만 맞으면 바로 꽂힙니다.

## 3. 캐릭터 성장 서사 — `보류 (재설계 필요)`

- **왜 보류인가**: 기존 "OH... ENGLESH?" 팻말 문구도 사용자 요청으로 현재 렌더 안 함(로직·CSS는 보존). 캐릭터 아트(항목 2)가 확정된 뒤, 성장 서사 자체를 다시 논의할 필요가 있음 — 유령 전용 "NO ENGLISH→OK ENGLISH" 서사는 캐릭터가 4종 범용화되면서 그대로 쓰기 애매해짐.
- 캐릭터 아트 확정 후 별도로 방향 논의 예정.

## 4. 뱃지 아이콘 12종 (뱃지 컬렉션) — `여유 있음 (로직 확정 후)`

- **왜 후순위인가**: 뱃지 획득 조건(로직) 자체가 아직 미정 — 12개 카테고리가 뭐가 될지부터 정해야 아이콘 컨셉도 나옴.
- **스펙(확정되면)**: 정사각형 아이콘 12개, 탭하면 뒤집히는(flip) 연출 — 앞면(아이콘)만 디자인, 뒷면은 텍스트 처리. 색상은 형광펜 하이어라키 규칙(위 "완료" 섹션 참고)을 그대로 따름 — 연한 배경 + 가장진한 텍스트/아이콘 색 조합.
- **선행 필요**: 로직 논의를 먼저 하고 싶으시면 말씀해 주세요.

## 5. 페르소나 아바타 10종 (회화 기능) — `회화 기능 착수 전까지`

- **왜 후순위인가**: 회화 탭 자체가 아직 미착수라 지금 당장은 안 급함.
- **스펙**: 2D 아바타, 정사각형, 캐릭터 세트(항목 2)와 같은 스타일(형광펜 색 채움 + 러프닝 아웃라인)로 통일. 페르소나 목록(PLAN.md 4.6 표)에 이름·속성이 이미 정의되어 있음.

## 6. 최종 전체 디자인 시안 · 실기기 검수 — `마감 직전`

- 위 1~5가 다 반영된 뒤 최종적으로 전체 화면 일관성 점검 + 실기기(iOS PWA) 테스트.

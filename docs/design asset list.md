# 사용자 준비 디자인 에셋 목록

작성일: 2026-07-12 · 기준: PLAN.md 실제 구현 현황(0장)과 대조해 급한 순서로 정렬

이 프로젝트는 고정 출시일이 없는 개인용 앱이라 "시한"은 달력 날짜가 아니라 **그 에셋이 없으면 막히는 작업 시점** 기준입니다. 급한 순서대로 정렬했습니다.

---

## 1. PWA 앱 아이콘 — `지금 바로`

- **왜 1순위인가**: `manifest.json`이 이미 `img/icon-192.png`, `img/icon-512.png`를 참조하는데 실제 파일이 없음. 지금 홈 화면에 추가하면 아이콘이 안 뜸 — 이미 완성된 다른 기능들의 체감 완성도를 깎아먹는 가장 저렴하고 빠르게 고칠 수 있는 구멍.
- **스펙**:
  - `img/icon-192.png` — 192×192px, PNG
  - `img/icon-512.png` — 512×512px, PNG (같은 디자인을 배율만 다르게)
  - (선택, iOS 홈 화면 품질용) `img/apple-touch-icon.png` — 180×180px, 배경색 있어야 함(투명 배경 X, iOS가 둥근 사각형으로 자동 마스킹)
- **컨셉 제안**: 메인 캐릭터(유령)나 앱 심볼(NO ENGLISH 팻말 등) 단순화한 아이콘. 배경은 `manifest.json`의 `background_color`(#f5f6f8)와 어울리게.

## 2. 디자인 토큰 확정 (컬러 · 타이포) — `지금 바로`

- **왜 2순위인가**: 원래 1주차에 확정하기로 했던 항목인데 아직 임시값(플레이스홀더)으로 개발이 계속 진행 중. `index.html` 상단에 "포인트 컬러(--accent)는 임시 플레이스홀더. 디자이너가 교체할 자리"라고 코드에도 명시되어 있음. 이게 늦어질수록 나중에 만드는 화면(회화 채팅 UI 등)도 임시색으로 계속 쌓여서 나중에 한꺼번에 바꾸는 리스크가 커짐.
- **파일 형태**: 별도 이미지 파일이 아니라 **값 목록**이면 충분 (Figma 링크, 또는 텍스트로 hex 코드만 전달해도 됨):
  - 포인트 컬러(`--accent`) 1개 — 토스 파랑은 피하기로 함(COMMERCIAL.md 차별화 기준)
  - 배경/서피스/텍스트 톤(라이트 모드 온리)
  - 폰트 패밀리(한글+영문) — 상업 배포 가능한 라이선스(OFL 등) 확인 필요
- **적용 위치**: `index.html` CSS 변수(`:root` 또는 상단 스타일 블록)에 제가 바로 대입 가능.

## 3. 캐릭터 러닝 스프라이트 — 고양이 유령 · 두더지 · 외계인 (3종) — `이번 주 안`

- **왜 3순위인가**: 홈 화면은 앱을 열 때마다 보는 핵심 화면인데, 현재 4종 캐릭터 중 **쥐만 실제 그림**이고 나머지 3종은 텍스트 placeholder("두더지 / 땅 파며 질주 / 2단계" 식 글자)로 뜸. 코드 슬롯은 이미 완성되어 있어서 그림만 들어오면 바로 반영됨.
- **참고 패턴**: `img/rat_run.js`에 이미 있는 쥐 캐릭터 방식 그대로 — 프레임 개수·구조를 맞추면 코드 수정 없이 꽂힙니다.
- **스펙(캐릭터당)**:
  - 4프레임 SVG (달리기 사이클 1주기), 각 프레임 정사각형 viewBox
  - 벡터 원색+아웃라인 라인 스타일(5.2 확정 스타일)
  - 파일 전달 시 SVG 원본 그대로 주시면 svgo 최적화는 제가 처리
- **캐릭터별 동작 컨셉**(이미 기획에 정의됨):
  - 고양이 유령 — 스케이트보드 타기
  - 두더지 — 땅 파며 질주
  - 외계인 — UFO 타고 이동
- **우선순위 내에서도 순서 제안**: 셋 다 급하지만, 굳이 순서를 매기면 온보딩에서 가장 먼저 노출 가능성이 높은 캐릭터(유령 — 메인 캐릭터 후보)부터 하시면 됩니다.

## 4. 학습 카드 · 홈 화면 UI 시안 — `이번 주 안 (밀린 항목)`

- **왜 4순위인가**: 원래 "2주차 병행 작업"으로 잡혀 있던 항목인데(카드·홈 화면 시안 → 주말 1차 반영), 개발은 이미 2주차 내용(단어 네트워크)까지 진행돼서 사실 지금 시점에 받아야 밀리지 않음.
- **필요한 화면**: 학습 카드(빈칸 채우기 진행 중 화면), 홈 화면(현재 캐릭터+스트릭+뱃지 트랙 레이아웃의 최종 비주얼).
- **형태**: Figma 링크 또는 이미지 목업. 지금 있는 것(하드코딩 CSS)을 그대로 뜯어고쳐도 되니 "이 색·이 여백·이 폰트로 바꿔주세요" 수준의 지시여도 충분합니다.

## 5. 캐릭터 성장 스테이지 아트 (유령, 5컷) — `2주 내`

- **왜 5순위인가**: PLAN.md 4.10에서 "공수 대비 효과 최고"로 분류된 항목이라 순위상 뱃지·페르소나보다 앞에 둠. 다만 아직 홈 화면 자체가 완성 단계라 지금 당장 막히진 않음.
- **스펙**: 유령 캐릭터의 "NO ENGLISH" 팻말 문구/표정이 누적 학습량에 따라 5단계로 바뀌는 컷 — 예: `NO ENGLISH` → `OH... ENGLISH?`(현재 placeholder 문구) → ... → `OK ENGLISH` 식 서사. 각 컷은 독립 정적 이미지(러닝 스프라이트처럼 4프레임 애니메이션까진 필요 없음, 이미지 교체만 하면 됨).
- 개발 측 로직(이미지 스왑)은 이미 준비 완료 — 그림만 있으면 바로 연결.

## 6. 뱃지 아이콘 12종 (뱃지 컬렉션) — `여유 있음 (로직 확정 후)`

- **왜 6순위인가**: 뱃지 획득 조건(로직) 자체가 아직 미정(세션 태스크 #19) — 12개 카테고리가 뭐가 될지부터 정해야 아이콘 컨셉도 나옴. 지금 그리시면 나중에 카테고리가 바뀌어 헛수고할 위험이 있음.
- **스펙(확정되면)**: 정사각형 아이콘 12개, 애플워치 뱃지 컬렉션처럼 탭하면 뒤집히는(flip) 연출이라 **앞면(아이콘)/뒷면(간단 문구+날짜) 구도**를 고려해서 앞면만 디자인하면 됨(뒷면은 텍스트로 처리).
- **선행 필요**: 로직 논의를 먼저 하고 싶으시면 말씀해 주세요.

## 7. 페르소나 아바타 10종 (회화 기능) — `회화 기능 착수 전까지`

- **왜 7순위인가**: 회화 탭(4.5/4.6) 자체가 아직 미착수라 지금 당장은 안 급함. 다만 10종이라 물량이 많아서 회화 기능 개발 착수 전에 여유 있게 시작하시는 걸 권장.
- **스펙**: 2D 아바타, 정사각형, 캐릭터 세트(5.2)와 같은 원색+라인 스타일로 통일. 페르소나 목록(PLAN.md 4.6 표)에 이름·속성이 이미 정의되어 있음 — IT회사 동료, 애인, 여행지 사람, 마트 점원, 공항 직원, 의사, 카페 직원, 행인, 튜터, 10년지기 친구.
- 사용자가 직접 만드는 "내 페르소나"는 아바타를 이 기본 캐릭터 세트(항목 3, 5)에서 재사용하므로 별도 제작 불필요.

## 8. 최종 전체 디자인 시안 · 실기기 검수 — `4주차(마감 직전)`

- PLAN.md 6장 타임라인상 4주차 항목. 위 1~7이 다 반영된 뒤 최종적으로 전체 화면 일관성 점검 + 실기기(iOS PWA) 테스트.
